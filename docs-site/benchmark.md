# 性能数据

> 基于 Apache JMeter 5.6.3 全场景压测，依据 ISO/IEC 25010 性能效率模型与 Google SRE 四大黄金信号度量。

## 压测总览

| 关键指标 | 数值 | 评级 |
| :--- | ---: | :--- |
| 总样本数 | 1015 | — |
| **错误率（剔除业务校验异常路径）** | **0.00%** | ✅ 优秀 |
| 错误率（含业务校验拒绝） | 4.24% | 见下方说明 |
| 吞吐量 TPS | 34.71 req/s | 符合预期 |
| **响应时间中位数 P50** | **321 ms** | ✅ 优秀 |
| 响应时间 P95 | 2237 ms | 合格 |
| 响应时间 P99 | 3902 ms | 合格 |
| **APDEX 2.0 得分（T=500ms）** | **0.75** | 合格（Fair） |
| 测试场景覆盖率 | 4 / 4 | ✅ 100% |

::: tip 关于 4.24% 错误率
43 条错误全部发生在「学生提交考试」步骤，属于**异常路径压测的有效发现**——即对已交卷 / 重复提交等非法状态的拒绝。剔除这些业务校验拒绝后，**系统级错误率为 0.00%**，无任何 500 / 超时 / 连接异常。
:::

### 三重环境隔离

按 ISO/IEC/IEEE 29119-3 §8.2.4「测试环境」要求，perf 与 dev 严格隔离：

1. **数据库层**：独立 Schema，与 dev 零重叠
2. **缓存层**：Redis `database: 1`，独立 key 空间
3. **配置层**：`application-perf.yml` 覆盖连接池、关闭 SQL 日志、开放 Actuator 端点

## 生产级配置保障

性能数据背后是一套接近生产级的配置：

| 配置项 | 值 | 作用 |
| :--- | :--- | :--- |
| HikariCP 连接池 | max=20, min-idle=5, max-lifetime=30min | 生产环境数据库连接管理 |
| Tomcat 线程池 | max=200, min-spare=20, accept-count=100 | 高并发请求处理 |
| 优雅停机 | `server.shutdown: graceful` (30s) | 无损发布，避免请求中断 |
| 日志滚动 | 按天 + 按大小，保留 30 天 / 1GB | 生产日志治理 |
| 登录限流 | Nginx 5r/m + Guava 应用层 | 防 brute force |
| 安全响应头 | OWASP 推荐全量配置 | XSS / 点击劫持防护 |

::: details 度量依据
- **ISO/IEC/IEEE 29119-3:2021** §8.2.5 测试执行报告、§8.2.6 测试完成报告
- **ISO/IEC 25010:2011** §4.2 性能效率（时间特性 / 资源利用率 / 容量）
- **Google SRE** 四大黄金信号（延迟 / 流量 / 错误 / 饱和度）
- **APDEX 2.0**（T=500ms）用户满意度指数
:::
