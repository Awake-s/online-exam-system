## 1. 架构概览

```
                    ┌─────────────┐
    用户浏览器 ───→ │  Nginx :80   │
                    │  /:443      │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        静态文件        /api/*      /ws/*
        (Vue dist)     反向代理    WebSocket代理
              │            │            │
              │     ┌──────┴──────┐     │
              │     │ Spring Boot │     │
              │     │   :8081     │     │
              │     └──────┬──────┘     │
              │       ┌────┴────┐       │
              │   MySQL:3306  Redis:6379│
              │    (本地)      (本地)    │
              └─────────────────────────┘
```

**关键设计**：
- Nginx 同时托管前端静态文件 + 反向代理后端 API
- MySQL/Redis 只监听 `127.0.0.1`，不暴露到公网
- 后端 8081 端口不对外开放，只通过 Nginx 转发

## 3. 域名与备案

### 3.1 ICP 备案（中国大陆必须）

在中国大陆，网站绑定域名必须完成 ICP 备案，否则会被运营商拦截。

**流程**：
1. **购买域名**：在云厂商控制台购买（与服务器同一厂商最方便）
2. **实名认证**：上传身份证，1~3 个工作日
3. **提交备案**：在云厂商「备案中心」提交
   - 首次备案：~15~20 个工作日
   - 新增网站：~7~10 个工作日
4. **公安备案**：ICP 通过后 30 天内完成

**所需材料**：
- 身份证正反面照片
- 手机号
- 域名证书（云厂商控制台下载）
- 网站名称（如"在线考试系统"）
- 网站用途描述

### 3.2 不需要备案的情况

- 使用 IP 直接访问（不绑定域名）
- 服务器在香港/海外
- 仅局域网内部使用

## 6. 部署后端

### 6.1 上传文件到服务器

从本地 Windows 执行（或用 WinSCP/SFTP 工具）：

```powershell
# 上传 JAR 包
scp "D:\Java Projects\在线考试系统\exam-system\target\exam-system-1.0.0.jar" root@服务器IP:/opt/exam-system/

# 上传数据库脚本
scp "D:\Java Projects\在线考试系统\exam-system\sql\online_exam_system.sql" root@服务器IP:/opt/exam-system/

# 上传 systemd 服务文件
scp "D:\Java Projects\在线考试系统\deploy\04-运行配置\exam-system.service" root@服务器IP:/etc/systemd/system/
```

### 6.2 初始化数据库

```bash
# 导入全量 SQL
mysql -u root -p'你的MySQL根密码' online_exam_system < /opt/exam-system/online_exam_system.sql
```

### 6.3 配置 systemd 服务

编辑服务文件中的密码：

```bash
sudo nano /etc/systemd/system/exam-system.service
# 修改 Environment= 行中的密码
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable exam-system
sudo systemctl start exam-system

# 查看状态
sudo systemctl status exam-system

# 查看日志
sudo journalctl -u exam-system -f
```

### 6.4 验证后端

```bash
curl http://localhost:8081/actuator/health
# 应返回 {"status":"UP","components":{"db":{"status":"UP"},"redis":{"status":"UP"}}}
```

## 10. 安全加固

### 10.1 必做项

```bash
# 1. 修改 SSH 端口（防暴力扫描）
sudo nano /etc/ssh/sshd_config
# Port 22 → Port 2222（或其他端口）
sudo ufw allow 2222/tcp
sudo systemctl restart sshd

# 2. 禁用 root SSH 登录
# PermitRootLogin no（在 sshd_config 中设置）

# 3. 创建普通用户
adduser deploy
usermod -aG sudo deploy

# 4. MySQL 只监听本地
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
# bind-address = 127.0.0.1（默认已是）

# 5. Redis 设置密码 + 只监听本地
sudo nano /etc/redis/redis.conf
# requirepass YourRedisPassword
# bind 127.0.0.1 ::1
```

### 10.2 生产环境密码要求

| 配置项 | 说明 |
|--------|------|
| MySQL root 密码 | 修改为强密码，不要用 `12345678` |
| MySQL 应用用户 | 创建专用 `exam_app` 用户，只授权 `online_exam_system` 库 |
| Redis 密码 | 必须设置 `requirepass` |
| JWT Secret | 至少 256 位随机字符串 |
| Spring Boot Actuator | 生产环境只暴露 `health`，隐藏详情 |

### 10.3 CORS 限制

生产环境建议将 `WebMvcConfig.java` 中的 `allowedOriginPatterns("*")` 改为具体域名：

```java
.allowedOriginPatterns("https://exam.yourdomain.com")
```

WebSocket 配置同理。

## 12. 常见问题

### Q1: 访问网站显示 502 Bad Gateway
**原因**：后端未启动或端口不对
```bash
sudo systemctl status exam-system
curl http://localhost:8081/actuator/health
```

### Q2: WebSocket 连接失败
**原因**：Nginx 未配置 WebSocket 代理
- 检查 nginx.conf 中 `/ws` 和 `/ws/notification` 的 `proxy_pass` 和 `Upgrade` 头

### Q3: 上传文件后看不到
**原因**：`/opt/exam-system/uploads/` 权限问题
```bash
chown -R exam:exam /opt/exam-system/uploads/
```

### Q4: 数据库连接被拒绝
```bash
# 检查 MySQL 是否运行
sudo systemctl status mysql
# 检查用户权限
mysql -u exam_app -p'密码' -e "SHOW DATABASES;"
```

### Q5: 如何更新版本？
```bash
# 1. 本地构建新版本
# 2. 上传新的 JAR 和 dist
scp exam-system-1.0.0.jar root@server:/opt/exam-system/
scp -r dist/* root@server:/opt/exam-system/frontend/dist/

# 3. 重启后端
sudo systemctl restart exam-system

# 4. Nginx 无需重启（静态文件直接替换生效）
```

### Q6: 如何查看实时日志？
```bash
# 后端
sudo journalctl -u exam-system -f

# Nginx 访问日志
sudo tail -f /var/log/nginx/access.log
```

## 附录 A：生产密码生成最佳实践

### A.1 密码强度要求

| 用途 | 最小长度 | 推荐长度 | 字符集 |
|------|---------|---------|--------|
| MySQL root | 16 | 24 | 字母 + 数字 + 符号 |
| MySQL 应用用户 | 16 | 24 | 字母 + 数字 |
| Redis | 16 | 32 | 字母 + 数字 |
| JWT Secret | 32 字节（256 位）| 48 字节 | Base64 |

### A.2 生成命令

```bash
# 数据库/Redis 密码（24 字符，去除易混字符）
openssl rand -base64 24 | tr -d '=/+' | head -c 24

# JWT Secret（48 字节 Base64，约 64 字符）
openssl rand -base64 48 | tr -d '\n'

# 一次生成所有密码并保存
cat > /tmp/all-passwords.txt <<EOF
MySQL root:    $(openssl rand -base64 24 | tr -d '=/+' | head -c 24)
App DB:        $(openssl rand -base64 24 | tr -d '=/+' | head -c 24)
Redis:         $(openssl rand -base64 24 | tr -d '=/+' | head -c 24)
JWT Secret:    $(openssl rand -base64 48 | tr -d '\n')
EOF
chmod 600 /tmp/all-passwords.txt
cat /tmp/all-passwords.txt
```

### A.3 密码管理原则

- ❌ **绝对禁止**：密码写在代码、Git、Dockerfile、docker-compose.yml 里
- ❌ **绝对禁止**：同一密码跨服务复用（DB 和 Redis 用一样）
- ❌ **绝对禁止**：用"记得住的"密码（生日、姓名、公司名）
- ✅ **推荐**：所有密码通过 `/etc/exam-system/secrets.env` 注入，权限 600
- ✅ **推荐**：密码管理器（1Password / Bitwarden）保存凭据
- ✅ **推荐**：定期轮换（每 90 天），轮换后 `systemctl restart exam-system`

## 附录 C：CORS / WebSocket 白名单精确化

### C.1 生产环境必须精确域名（代码已支持）

已通过 `@Value("${app.cors.allowed-origins:*}")` 实现配置驱动：

```yaml
# application-prod.yml
app:
  cors:
    allowed-origins: ${APP_CORS_ORIGINS:https://kimi888.xyz,https://kimi123.site}
```

```bash
# /etc/exam-system/secrets.env（线上覆盖）
APP_CORS_ORIGINS=https://kimi888.xyz,https://www.kimi888.xyz,https://kimi123.site
```

### C.2 多域名场景配置

```bash
# 双轨部署：香港 + 国内同时生效
APP_CORS_ORIGINS=https://kimi888.xyz,https://www.kimi888.xyz,https://kimi123.site,https://kimi666.online
```

### C.3 验证 CORS 是否生效

```bash
# 模拟预检请求（OPTIONS）
curl -I -X OPTIONS \
     -H "Origin: https://kimi888.xyz" \
     -H "Access-Control-Request-Method: POST" \
     https://kimi888.xyz/api/auth/login
```

## 附录 E：应急回滚预案

### E.1 部署前备份当前版本

```bash
# 在上传新 JAR 前备份旧版本
VERSION_TAG="v$(date +%Y%m%d_%H%M%S)"
sudo cp /opt/exam-system/exam-system-1.0.0.jar \
       /opt/exam-system/backups/exam-system-${VERSION_TAG}.jar

# 备份当前前端
sudo tar -czf /opt/exam-system/backups/frontend-${VERSION_TAG}.tar.gz \
       -C /opt/exam-system/frontend dist
```

### E.2 回滚到上一个版本

```bash
# 列出可回滚的版本
ls -lt /opt/exam-system/backups/ | head -10

# 回滚后端（示例）
sudo systemctl stop exam-system
sudo cp /opt/exam-system/backups/exam-system-v20260422_143000.jar \
       /opt/exam-system/exam-system-1.0.0.jar
sudo systemctl start exam-system

# 回滚前端
sudo rm -rf /opt/exam-system/frontend/dist
sudo tar -xzf /opt/exam-system/backups/frontend-v20260422_143000.tar.gz \
       -C /opt/exam-system/frontend/

# 回滚数据库（谨慎！只在绝对必要时）
gunzip < /opt/exam-system/backups/mysql/db_20260422.sql.gz \
    | mysql -u root -p online_exam_system
```

### E.3 蓝绿部署（零停机升级）

```bash
# 1. 在 8082 端口启动新版本
cp exam-system-1.0.1.jar /opt/exam-system/exam-system-new.jar
java -jar /opt/exam-system/exam-system-new.jar \
     --server.port=8082 \
     --spring.profiles.active=prod &

# 2. 验证新版本健康
curl http://localhost:8082/actuator/health

# 3. 切换 Nginx 上游
sudo sed -i 's/127.0.0.1:8081/127.0.0.1:8082/' \
     /etc/nginx/sites-available/exam-system
sudo nginx -t && sudo systemctl reload nginx

# 4. 停止旧版本
sudo systemctl stop exam-system
```
