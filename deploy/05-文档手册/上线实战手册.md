### 3.8 部署 Nginx（先走 HTTP，后续升 HTTPS）

```bash
# 部署 Nginx 配置（HTTP 版）
sudo cp /root/nginx.conf /etc/nginx/sites-available/exam-system

# 关键：把限流 zone 放到 http 块
sudo tee /etc/nginx/conf.d/00-exam-ratelimit.conf > /dev/null <<'EOF'
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=20r/s;
limit_req_zone $binary_remote_addr zone=login_limit:10m rate=5r/m;

log_format exam_access '$remote_addr - $remote_user [$time_local] '
                       '"$request" $status $body_bytes_sent '
                       '"$http_referer" "$http_user_agent" '
                       'rt=$request_time uct="$upstream_connect_time" '
                       'uht="$upstream_header_time" urt="$upstream_response_time"';
EOF

sudo ln -sf /etc/nginx/sites-available/exam-system /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# 测试配置 + 重载
sudo nginx -t
sudo systemctl reload nginx
```

### 3.9 申请 Let's Encrypt SSL 证书

**前置条件**：
- DNS 已解析（`nslookup kimi123.site` 能返回 IP）
- 80 端口开放
- 服务器已装 certbot（deploy.sh 已装）

```bash
# 一键申请（自动改写 Nginx 配置）
sudo certbot --nginx \
    -d kimi123.site \
    -d www.kimi123.site \
    --email your@email.com \
    --agree-tos --no-eff-email \
    --redirect

# 测试续期
sudo certbot renew --dry-run

# 查看续期 timer
systemctl list-timers | grep certbot
```

**⚠️ certbot 会修改 Nginx 配置**。申请成功后，建议用我们更完整的 `nginx-https.conf` 替换：

```bash
# 备份 certbot 改过的配置
sudo cp /etc/nginx/sites-available/exam-system /etc/nginx/sites-available/exam-system.certbot.bak

# 替换为我们的 HTTPS 版（需要先把 server_name 改为 kimi123.site）
sudo sed -i 's/kimi888.xyz/kimi123.site/g' /root/nginx-https.conf
sudo sed -i 's/www.kimi123.site www.kimi123.site/www.kimi123.site/' /root/nginx-https.conf
sudo cp /root/nginx-https.conf /etc/nginx/sites-available/exam-system

# 测试 + 重载
sudo nginx -t
sudo systemctl reload nginx

# 访问验证
# https://kimi123.site  ← HTTPS 锁图标 ✅
```

### 6.3 版本升级流程

**零停机升级**（推荐）：

```bash
# 在本机构建新版本
mvn clean package -DskipTests
npm run build

# 上传新 JAR（命名带版本号）
scp exam-system-1.0.0.jar root@<IP>:/opt/exam-system/exam-system-1.0.1.jar

# 在服务器上备份旧版本
sudo cp /opt/exam-system/exam-system-1.0.0.jar \
       /opt/exam-system/backups/exam-system-$(date +%Y%m%d_%H%M%S).jar

# 停老 → 换 → 启新（实际停机 5-10 秒）
sudo systemctl stop exam-system
sudo mv /opt/exam-system/exam-system-1.0.1.jar /opt/exam-system/exam-system-1.0.0.jar
sudo systemctl start exam-system

# 验证
curl http://localhost:8081/actuator/health
```

**前端更新**（零停机）：

```bash
# 直接替换 dist 目录
scp -r dist/* root@<IP>:/opt/exam-system/frontend/dist/
# 无需重启任何服务（静态文件替换即生效）
```


### A.2 被 DDoS / CC 攻击

```bash
# 查看当前连接
sudo ss -ant | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn | head

# 封禁高频 IP（示例）
sudo ufw deny from <恶意IP>

# 开启 Nginx 限流（已默认开启）
# 确认限流生效：tail /var/log/nginx/exam-error.log | grep limiting
```

### A.3 数据库误删恢复

```bash
# 找到最近的备份
ls -lt /opt/exam-system/backups/mysql/ | head

# 停应用
sudo systemctl stop exam-system

# 恢复
gunzip < /opt/exam-system/backups/mysql/db_20260423_030000.sql.gz \
    | mysql -u root -p online_exam_system

# 启动应用
sudo systemctl start exam-system
```

### A.4 SSL 证书过期

```bash
# 手动续期
sudo certbot renew --force-renewal
sudo systemctl reload nginx

# 排查为什么自动续期没工作
systemctl list-timers | grep certbot
sudo journalctl -u certbot.timer
```
