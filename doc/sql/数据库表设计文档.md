## 表1：角色表 (sys_role)

**表说明**：存储系统的三种角色：管理员(ADMIN)、教师(TEACHER)、学生(STUDENT)

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 角色ID |
| role_name | VARCHAR(50) | NOT NULL | - | 角色名称 |
| role_code | VARCHAR(50) | NOT NULL, UNIQUE | - | 角色编码：ADMIN/TEACHER/STUDENT |
| description | VARCHAR(255) | - | NULL | 角色描述 |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |

**初始数据**：
| id | role_name | role_code | description |
|----|-----------|-----------|-------------|
| 1 | 管理员 | ADMIN | 系统管理员 |
| 2 | 教师 | TEACHER | 教师 |
| 3 | 学生 | STUDENT | 学生 |

## 表6：题目表 (exam_question)

**表说明**：存储题库中的所有题目，支持5种题型

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 题目ID |
| subject_id | BIGINT | NOT NULL, FK → edu_subject.id | - | 所属科目ID |
| question_type | TINYINT | NOT NULL | - | 题型：1单选 2多选 3判断 4填空 5简答 |
| content | MEDIUMTEXT | NOT NULL | - | 题目内容（最长 16MB，支持富文本/图片 base64） |
| options | JSON | - | NULL | 选项（JSON格式，选择题使用） |
| answer | MEDIUMTEXT | NOT NULL | - | 正确答案（兼容简答题长答案） |
| analysis | MEDIUMTEXT | - | NULL | 答案解析（兼容长解析文本） |
| score | DECIMAL(5,2) | - | 0 | 默认分值 |
| difficulty | TINYINT | - | 1 | 难度：1简单 2中等 3困难 |
| creator_id | BIGINT | NOT NULL, FK → sys_user.id | - | 创建人ID（教师） |
| deleted | TINYINT | - | 0 | 是否删除：0正常 1已删除 |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | ON UPDATE | CURRENT_TIMESTAMP | 更新时间 |

**题型说明**：
| question_type | 类型名称 | 选项格式 | 答案格式 |
|:-------------:|----------|----------|----------|
| 1 | 单选题 | ["A.选项1","B.选项2","C.选项3","D.选项4"] | "A" |
| 2 | 多选题 | ["A.选项1","B.选项2","C.选项3","D.选项4"] | "A,B,C" |
| 3 | 判断题 | - | "1"正确 / "0"错误 |
| 4 | 填空题 | - | "答案1,答案2" |
| 5 | 简答题 | - | 参考答案文本 |

## 表16：系统通知表 (sys_notification)

**表说明**：存储系统通知消息，支持多种通知类型

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 通知ID |
| user_id | BIGINT | NOT NULL, FK → sys_user.id | - | 接收者用户ID |
| type | VARCHAR(50) | NOT NULL | - | 通知类型 |
| title | VARCHAR(200) | NOT NULL | - | 通知标题 |
| content | VARCHAR(500) | - | NULL | 通知内容详情 |
| biz_type | VARCHAR(50) | - | NULL | 关联业务类型: exam/score/user |
| biz_id | BIGINT | - | NULL | 关联业务ID |
| is_read | TINYINT | - | 0 | 是否已读: 0未读 1已读 |
| **priority** | **TINYINT** | **NOT NULL** | **2** | **E1 新增：优先级 1=紧急 / 2=普通 / 3=次要，UI 差异化渲染** |
| **payload** | **JSON** | **-** | **NULL** | **E1 新增：扩展载荷 JSON，承载 senderId/senderName/senderAvatar/actionUrl/extras** |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |

**通知类型**（全量 12 种，与 `NotificationTypeWhitelist` 白名单严格一致）：

考试相关 8 种：EXAM_PUBLISHED / EXAM_UPDATED / EXAM_CANCELLED / EXAM_CREATED / EXAM_SUBMITTED / EXAM_AUTO_SUBMITTED / EXAM_ABSENT / EXAM_END_SUMMARY  
成绩相关 2 种：SCORE_PUBLISHED（首次发布）/ SCORE_UPDATED（教师修正后重新发布，10 分钟内合并）  
账号相关 2 种：ACCOUNT_CREATED（新用户欢迎）/ USER_CREATED（管理员收到的新用户创建通知）

**索引**（通知查询性能关键，缺失会导致未读统计/列表分页退化为全表扫）：
- `idx_noti_user_read` - (user_id, is_read, create_time DESC) 用户未读通知分页（最常用）
- `idx_noti_user_time` - (user_id, create_time DESC) 用户全部通知按时间倒序
- `idx_noti_user_type_biz` - (user_id, type, biz_id) 按用户+类型+业务ID定位（幂等去重/冲突检测）
- `idx_noti_user_type_read_time` - **E1 新增**：(user_id, type, is_read, create_time DESC) 通知中心「类型+已读」组合筛选的覆盖索引，消除 filesort
- `idx_noti_read_time` - (is_read, create_time) 系统级未读统计/清理任务

**E1 升级说明**（2026-04-24）：
- 新增 `priority` / `payload` 两列：优先级差异化（紧急通知红色强调 + 强制弹窗/响铃豁免），payload 承载发送者头像/姓名和 action_url 路由
- 新增 `idx_noti_user_type_read_time` 覆盖索引：通知中心"按类型筛选 + 按已读筛选"查询走索引无需回表，大数据量下性能提升显著
- 旧通知数据兼容：`priority DEFAULT 2`、`payload DEFAULT NULL`，E1 升级前的历史通知零改造自动适配

## 表17：聊天会话表 (chat_conversation)

**表说明**：存储两个用户之间的一对一聊天会话，user1_id 始终小于 user2_id 以确保唯一性。支持用户级单向隐藏（user?_hidden），对方再发消息时自动解除隐藏。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 会话ID |
| user1_id | BIGINT | NOT NULL, FK → sys_user.id | - | 用户1（较小ID） |
| user2_id | BIGINT | NOT NULL, FK → sys_user.id | - | 用户2（较大ID） |
| last_message | VARCHAR(500) | - | NULL | 最后一条消息摘要 |
| last_message_time | DATETIME | - | NULL | 最后消息时间 |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| user1_hidden | TINYINT | - | 0 | **L3** 用户1是否隐藏该会话（0否 1是）；对方再发消息时自动重置为 0 |
| user2_hidden | TINYINT | - | 0 | **L3** 用户2是否隐藏该会话（0否 1是）；对方再发消息时自动重置为 0 |

**约束**：
- `uk_users` - (user1_id, user2_id) 联合唯一索引

## 表18：聊天消息表 (chat_message)

**表说明**：存储聊天消息详情，支持已读状态标记。**L3** 消息使用软删（`deleted_at IS NOT NULL`），保留审计轨迹；查询端需过滤或渲染为“已撤回”占位。

| 字段名 | 数据类型 | 约束 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | - | 消息ID |
| conversation_id | BIGINT | NOT NULL, FK → chat_conversation.id | - | 所属会话ID |
| sender_id | BIGINT | NOT NULL, FK → sys_user.id | - | 发送者ID |
| receiver_id | BIGINT | NOT NULL | - | 接收者ID |
| content | MEDIUMTEXT | NOT NULL | - | 消息内容（mediumtext，支持长消息/图片 base64 内嵌） |
| message_type | TINYINT | - | 1 | 消息类型：1文字 2图片(预留) |
| is_read | TINYINT | - | 0 | 是否已读：0未读 1已读 |
| create_time | DATETIME | - | CURRENT_TIMESTAMP | 发送时间 |
| deleted_at | DATETIME | - | NULL | **L3** 软删时间，NULL=未删除；非NULL=已撤回/管理员强删 |
| deleted_by | BIGINT | - | NULL | **L3** 删除人用户ID（发送者撤回=自身；管理员强删=管理员ID） |

**索引**：
- `idx_conv_time` - (conversation_id, create_time) 会话内消息时序翻页
- `idx_receiver_read` - (receiver_id, is_read) 未读消息计数/查询
- `idx_deleted` - (deleted_at) **L3** 快速过滤已删除消息
