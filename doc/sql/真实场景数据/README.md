## 五、执行顺序

### 一键执行（Windows PowerShell）

```powershell
$base = "d:\Java Projects\在线考试系统\doc\sql\真实场景数据"
foreach ($f in '01_基础数据','01b_专业扩容_107至127','05_班级扩容_22级与24级','06_学科表升级_年级专业课程化','07_课程数据_22级与24级真实课表','02_题库') {
    Copy-Item "$base\$f.sql" "$env:TEMP\$f.sql" -Force
    cmd /c "mysql -uroot -p12345678 --default-character-set=utf8mb4 online_exam_system < %TEMP%\$f.sql"
}
```

> **⚠ 关键依赖**：`01b_专业扩容_107至127.sql` 必须在 `05_班级扩容` 之前执行。  
> `01_基础数据` 只写入 7 个专业 (100-106)，而班级/课程脚本引用 major_id 107-127；  
> 跳过 01b 会导致 `ERROR 1452 fk_class_major` 外键失败。

⚠️ **重要**：Windows PowerShell 下管道会破坏中文编码，必须用 `cmd /c "mysql ... < file.sql"` 的字节级重定向，不要用 `Get-Content | mysql` 管道。

**教师可通过 UI 题库管理「Excel 导入」直接上传 Excel 增量加题，无需手动跑脚本。**

## 六、设计原则

| 原则 | 实施细节 |
|---|---|
| **零侵入** | 全部新增 ID 从 100 起步，不影响 admin (id=1) / taozhan (id=8) / luweizhong (id=14) 等核心账号 |
| **可回滚** | 每个 SQL 文件顶部 `START TRANSACTION`，末尾 `COMMIT`，出错可整体回滚 |
| **可重复执行** | 每个文件顶部 `DELETE WHERE id >= 100` 清理后再 INSERT，反复执行结果一致 |
| **可验证** | 每个文件末尾 `SELECT COUNT(*) ... UNION ALL ...` 断言计数 |
| **密码统一** | 所有新教师/学生密码均为 `123456`（BCrypt 哈希与现有 taozhan 一致） |
| **真实姓名** | 490 个学生 + 24 个教师全部使用真实中国姓名，性别按专业合理分布（工科 7:3，日语 2:8） |
| **多对多任课** | v2.0 从 1:1 班主任模型升级为多对多任课：公共课老师 1 人教 4-6 班，专业核心课 1 人教 1-2 个平行班 |
| **学号规则**（v2.1） | 学生学号 = YY(2) + 专业招生代码(2) + 1(本科) + 学院代码(2) + 班号(1) + 班内序号(2)。例：`2430107203` = 24 级计算机 2 班 03 号。专业代码取自学校 2025 江苏招生计划表（计30 通32 信29 机33 电31 土28 日10） |
| **工号规则**（v2.1 设计 / v2.4 仅作知识保留） | 教师工号 = YY 入职年(2) + 学院代码(2) + 学院内序号(2)。例：`080201` = 08 年入职 / 02 数理学院 / 第 1 号（王建华）。⚠ v2.4 起 username **不再使用工号**，改为拼音；该编码规则作为"真实学校人事系统编码"知识保留在 P1 SQL 注释中 |
| **登录账号规则**（v2.4） | **教师 username = 姓名拼音**（与邮箱前缀一致，如 `wangjianhua` ↔ `wangjianhua@usts-tp.edu.cn`）；**学生 username = 10 位学号**（与教务系统一致，如 `2430107203` ↔ `2430107203@stu.usts-tp.edu.cn`）。两者均符合真实苏科大「邮箱前缀 / 学号」二选一登录的统一身份认证习惯 |
| **学院代码** | 02 数理 / 03 城建 / 04 外语 / 05 语言文化 / 07 电子信息 / 08 智能制造与电气工程 |
| **题库唯一真相源**（v2.3 / v2.5 路径更新） | `doc/题库Excel/课程300题/` 是 **14 个 Excel** 的统一题源，与教师后台「题库管理 → Excel 导入」UI 完全兼容。Python 脚本 `工具_构建题库SQL.py` 仅负责解析 Excel 生成 SQL，不再依赖任何 JSON |
| **业务空始**（v2.2） | 教师/学生后台业务表（试卷/模板/考试/记录/答题）同始状态为空，由教师从题库手动创建模板/试卷/考试，以获得「从零到一」的完整业务闭环体验 |
