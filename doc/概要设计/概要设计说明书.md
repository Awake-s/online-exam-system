## 一、引言

### 1.1 编写目的

本文档是在线考试系统的概要设计说明书，旨在对系统的总体结构、模块划分、接口关系、数据结构以及运行环境进行全面阐述，为后续详细设计和编码实现提供顶层指导。

**预期读者**：项目开发人员、测试人员、系统架构师、项目管理人员、评审专家。

### 1.2 背景

- **系统名称**：在线考试系统（Online Exam System）
- **项目性质**：个人全栈技术实践项目
- **开发模式**：前后端分离（B/S架构）
- **应用场景**：以高校考试为业务原型的在线考试与教务管理演示系统，支持管理员、教师、学生三种角色
- **独立性说明**：本系统为独立的 Web 应用，不依赖外部第三方考试平台

### 2.3 设计约束

1. **前后端分离**：前端通过 RESTful API 与后端通信，前后端独立部署
2. **无状态认证**：采用 JWT 令牌，服务端不维护 Session 状态
3. **角色隔离**：前端路由按角色划分模块，后端接口通过 `@PreAuthorize` 注解控制权限
4. **数据隔离**：教师只能操作自己创建的资源（题目、试卷、考试），学生只能查看自己的数据
5. **统一响应格式**：所有 API 返回统一的 `{code, message, data}` JSON 结构
6. **逻辑删除**：部分表采用 `deleted` 字段实现软删除，由 MyBatis-Plus 全局配置

```

                              ↓
              学生进入考试 → 在线答题 → 提交试卷
                              ↓

                              ↓

                              ↓
                    系统通知学生 → 学生查看成绩
                              ↓
                    错题自动收集 → 学生复习
```

## 三、系统架构设计

### 3.1 架构风格

系统采用**分层架构（Layered Architecture）**与**前后端分离架构**相结合的设计模式：

| 层次 | 技术实现 | 职责 |
|------|----------|------|
| 展现层（Presentation） | Vue3 + Element Plus + Tailwind CSS | 用户界面渲染、交互、路由管理 |
| 接口层（API Gateway） | Axios + Vite Proxy | 请求封装、Token注入、错误拦截、代理转发 |
| 控制层（Controller） | Spring MVC @RestController | 请求接收、参数校验、权限校验、响应返回 |
| 业务层（Service） | Spring Service + @Transactional | 业务逻辑处理、事务管理、数据组装 |
| 持久层（Persistence） | MyBatis-Plus + Mapper XML | 数据库访问、SQL映射、分页查询 |
| 数据层（Data） | MySQL 8.0 | 数据持久化存储 |

#### 5.1.3 错误码定义

| 错误码 | 含义 | 触发场景 |
|:------:|------|----------|
| 200 | 成功 | 操作成功 |
| 400 | 参数错误 | 请求参数校验失败（@Valid） |
| 401 | 未认证 | Token无效或过期 |
| 403 | 无权限 | 角色权限不足 |
| 404 | 资源不存在 | 请求的数据不存在 |
| 409 | 业务冲突 | 重复操作、关联数据约束等 |
| 500 | 服务器错误 | 系统内部异常 |

### 6.1 概念结构设计（E-R模型）

```
┌─────────────────────────────────────────────────────────────────┐
│                      实体关系图（E-R图）                          │
└─────────────────────────────────────────────────────────────────┘

系统管理域：
    sys_role (1) ──────< (N) sys_user
    sys_notification ──→ sys_user (user_id)

教务管理域：
    edu_major (1) ─────< (N) edu_class (major_id)
    edu_class (1) ─────< (N) sys_user (class_id, 学生)
    edu_subject (M) >──< (N) edu_major（通过 subject_major 关联）
    sys_user (M) >─────< (N) edu_class（通过 teacher_class 关联，教师授课）
    sys_user (M) >─────< (N) edu_subject（通过 teacher_subject 关联，教师授课）

考试核心域：
    edu_subject (1) ───< (N) exam_question (subject_id)
    sys_user (1) ──────< (N) exam_question (creator_id, 教师)
    edu_subject (1) ───< (N) exam_paper (subject_id)
    sys_user (1) ──────< (N) exam_paper (creator_id, 教师)
    exam_question (M) >< (N) exam_paper（通过 exam_paper_question 关联）
    exam_paper_template (1) ──< (N) exam_template_rule
    exam_paper (1) ────< (N) exam_exam (paper_id)
    edu_class (1) ─────< (N) exam_exam (class_id)
    sys_user (1) ──────< (N) exam_exam (creator_id, 教师)
    exam_exam (1) ─────< (N) exam_record
    sys_user (1) ──────< (N) exam_record (user_id, 学生)
    exam_record (1) ───< (N) exam_answer
    exam_question (1) ─< (N) exam_answer (question_id)

即时通讯域：
    chat_conversation (1) ──< (N) chat_message
    sys_user (M) >─< (N) chat_conversation（user1_id, user2_id）
```

## 八、出错处理设计

### 8.1 出错处理策略

系统采用**全局统一异常处理**机制，通过 `@RestControllerAdvice` 注解的 `GlobalExceptionHandler` 捕获并处理所有异常：

```
异常发生
    │
    ▼
GlobalExceptionHandler 捕获
    │
    ├── MethodArgumentNotValidException → 400 参数校验失败
    ├── BusinessException               → 409 业务规则冲突
    ├── NotFoundException               → 404 资源不存在
    ├── AccessDeniedException           → 403 权限不足
    ├── AuthenticationException         → 401 认证失败
    └── Exception（兜底）               → 500 服务器内部错误 + 记录日志
    │
    ▼
返回统一 JSON 格式：{ code, message, data: null }
```

### 8.2 出错处理对策

| 错误类别 | 处理对策 | 用户感知 |
|----------|----------|----------|
| 参数校验错误 | 前端表单校验 + 后端 @Valid 双重校验 | 表单字段标红，提示具体错误信息 |
| 认证过期 | 前端拦截401响应，自动清除Token，跳转登录页 | 提示"登录已过期，请重新登录" |
| 权限不足 | 前端路由守卫 + 后端 @PreAuthorize 双重拦截 | 显示403无权限页面 |
| 业务冲突 | 后端校验并返回明确的业务错误描述 | ElMessage.error 弹出错误提示 |
| 资源不存在 | 后端返回404，前端显示404页面 | 显示"页面不存在" |
| 网络异常 | 前端 Axios 拦截器捕获网络错误 | 提示"网络错误，请检查网络连接" |
| 服务器内部错误 | 后端记录错误日志，返回500 | 提示"服务器异常，请稍后重试" |

### 8.3 删除操作关联检查

系统对所有删除操作实施**级联检查保护**，防止数据不一致：

| 删除对象 | 检查关联 | 有关联时 |
|----------|----------|----------|
| 用户（教师） | 题目、试卷、考试 | 拒绝删除 |
| 用户（学生） | 考试记录 | 拒绝删除 |
| 班级 | 学生、考试 | 拒绝删除 |
| 专业 | 班级 | 拒绝删除 |
| 学科 | 题目 | 拒绝删除 |
| 题目 | 试卷-题目关联 | 拒绝删除 |
| 试卷 | 考试 | 拒绝删除 |
| 考试 | 考试记录（已开始） | 拒绝删除 |

## 九、安全保密设计

### 9.1 认证安全

| 安全措施 | 实现方式 | 说明 |
|----------|----------|------|
| 密码加密存储 | BCryptPasswordEncoder | 不可逆哈希，自带盐值 |
| JWT无状态认证 | JJWT 0.11.5 | Token含userId、roleCode，有效期24小时 |
| Token黑名单 | TokenBlacklistService | 登出后Token立即失效，内存黑名单+定期清理 |
| 登录频率限制 | LoginRateLimiter | 5次失败锁定15分钟，防暴力破解 |
| 滑块验证 | 前端 ArtDragVerify 组件 | 人机验证，防自动化登录 |
| 图形验证码 | Hutool CaptchaUtil（可选） | 后端生成图形验证码作为增强机制 |

### 9.2 授权安全

| 安全措施 | 实现方式 | 说明 |
|----------|----------|------|
| 基于角色的访问控制 | @PreAuthorize("hasRole(...)") | 方法级权限注解 |
| 数据隔离 | SQL WHERE creator_id/user_id | 教师只操作自己的数据，学生只查看自己的 |
| 前端路由守卫 | Vue Router beforeEach | 角色路由隔离，非法访问重定向 |
| 接口权限矩阵 | SecurityConfig + @PreAuthorize | 19个Controller的完整权限配置 |

### 9.3 数据安全

| 安全措施 | 实现方式 | 说明 |
|----------|----------|------|
| XSS防护 | XssUtils 检测与转义 | 用户输入的关键字段进行XSS过滤 |
| CSRF防护 | 前后端分离 + JWT | 无状态架构天然免疫CSRF |
| SQL注入防护 | MyBatis-Plus 参数化查询 | 框架自动参数绑定 |
| 文件上传校验 | 白名单扩展名 + 大小限制 | 仅允许 jpg/png/gif/xlsx/xls |
| 考试数据保护 | 不返回答案和解析 | 学生考试时不暴露正确答案 |
| 点击劫持防护 | frameOptions().sameOrigin() | HTTP头限制iframe嵌入 |

### 9.4 考试防作弊

| 措施 | 前端实现 | 后端校验 |
|------|----------|----------|
| 全屏模式 | Fullscreen API 强制全屏 | — |
| 切屏检测 | visibilitychange + blur 事件 | recordSwitchScreen 原子递增计数 |
| 禁止复制粘贴 | CSS user-select:none + JS事件拦截 | — |
| 试题乱序 | — | shuffleWithSeed 稳定随机种子乱序 |
| 选项乱序 | — | relabelOptions 乱序后重标记ABCD |
| 切屏次数限制 | 超限自动提交 | anti_cheat_config 配置最大切屏次数 |
| 禁止重复考试 | — | exam_record 唯一约束(exam_id, user_id) |
| 无操作超时 | 鼠标/键盘事件监听 | — |
