## ⬜ 阶段三：P1 — 云原生 + 容器化

> **目标**：告别裸机部署，进入容器化与自动化交付。
>
> **总工时预估**：3-4 周

### P1-1 Spring Boot 3 + Java 17 升级

- [ ] JDK 切换到 17（`pom.xml` 的 `java.version` 改 17）
- [ ] `parent` 升级到 `spring-boot-starter-parent:3.2.x`
- [ ] 用 [OpenRewrite](https://docs.openrewrite.org/recipes/spring/boot3upgrade) 自动迁移 `javax.*` → `jakarta.*`
- [ ] `jjwt` 升级到 0.12.x（API 有 breaking change，需调整 `Jwts.builder()` 链式调用）
- [ ] 验证 MyBatis-Plus 3.5.5 兼容性（确认 ≥ 3.5.5 支持 Spring Boot 3）
- [ ] 验证 EasyExcel、Hutool、Guava 兼容性
- [ ] 验证 WebSocket（`HandshakeInterceptor` 等 API 变化）
- [ ] 跑全量回归，确保三端功能正常

**验收标准**：应用在 Java 17 + Spring Boot 3.2 下正常启动，三端功能回归通过。

**学习资源**：
- [Spring Boot 3.0 Migration Guide（官方）](https://github.com/spring-projects/spring-boot/wiki/Spring-Boot-3.0-Migration-Guide)
- [OpenRewrite Spring Boot 3 升级配方](https://docs.openrewrite.org/recipes/spring/boot3upgrade)

### P1-2 Docker 化

- [ ] 后端 `exam-system/Dockerfile`（多阶段：`maven:3.9-eclipse-temurin-17` 构建 → `eclipse-temurin:17-jre` 运行）
- [ ] 前端 `art-design-pro-ui/Dockerfile`（多阶段：`node:22-alpine` 构建 → `nginx:alpine` 运行）
- [ ] 根目录 `docker-compose.yml`：一键拉起 `mysql` + `redis` + `backend` + `frontend`
- [ ] `.dockerignore` 排除 node_modules / target / .git
- [ ] 验证 `docker compose up` 全栈可用

**验收标准**：`docker compose up -d` 后，访问前端能正常登录使用。

**学习资源**：[Spring Boot Docker 官方指南](https://spring.io/guides/topicals/spring-boot-docker)

### P1-3 配置中心化（Nacos）

- [ ] `docker-compose.yml` 加 Nacos 服务（单机模式）
- [ ] 引入 `spring-cloud-starter-alibaba-nacos-config`
- [ ] 把 `application-prod.yml` 的数据库/Redis/JWT 配置迁到 Nacos
- [ ] 用 `@RefreshScope` 支持配置热更新
- [ ] 本地 dev 保持 yml，prod 走 Nacos

**验收标准**：在 Nacos 控制台修改配置，应用无需重启即可生效。

> 注意：P1 阶段 Nacos 只做**配置中心**，注册中心留到 P2 拆微服务时启用。

### P1-4 持续交付

- [ ] 扩展 CI：构建 Docker 镜像 → 推到 GitHub Container Registry (GHCR)
- [ ] `.github/workflows/release.yml`：打 tag（如 `v1.0.0`）自动发版 + 生成 changelog
- [ ] 服务器 `docker compose pull && docker compose up -d` 一键更新

**验收标准**：打 tag 触发自动构建镜像并推到 GHCR；服务器一键拉取新版本。

## ⬜ 阶段四：P2 — 微服务拆分 + 可观测性（长期演进）

> **目标**：从单体演进到微服务，具备企业级高可用与可观测能力。
>
> **总工时预估**：6-8 周（可分多个迭代）

### P2-A 微服务化（Spring Cloud Alibaba）

- [ ] 引入 `spring-cloud-alibaba:2023.x`（对应 Spring Boot 3.2）
- [ ] 服务拆分（按业务域）：
  - [ ] `exam-gateway`：Spring Cloud Gateway（路由 / 限流 / 统一鉴权）
  - [ ] `exam-user-service`：用户 / 角色 / 权限 / 认证
  - [ ] `exam-question-service`：题库 / 试卷 / 组卷
  - [ ] `exam-exam-service`：考试 / 作答 / 阅卷 / 成绩
  - [ ] `exam-message-service`：聊天 / 通知
- [ ] **Nacos 注册中心**：服务注册与发现
- [ ] **Nacos 配置中心**：各服务独立配置命名空间
- [ ] **Sentinel**：网关流量控制、服务熔断降级、热点参数限流（如考试提交接口）
- [ ] **Seata**：分布式事务（组卷 + 考试提交等跨服务场景，AT 模式）
- [ ] **ShardingSphere**：考试记录 / 答题记录按班级分表

**验收标准**：5 个服务独立部署，通过 Gateway 统一访问；拆服务后业务功能与单体一致。

**学习资源**：
- [Spring Cloud Alibaba 官方文档](https://sca.aliyun.com/)
- [Nacos 官方文档](https://nacos.io/zh-cn/docs/what-is-nacos.html)
- [Sentinel 官方文档](https://sentinelguard.io/zh-cn/docs/introduction.html)

### P2-B 可观测性三件套

| 维度 | 方案 | 任务 |
|:---|:---|:---|
| 日志 | Loki + Promtail | [ ] 部署 Loki 栈；[ ] 各服务日志结构化（JSON）；[ ] Promtail 采集 |
| 指标 | Prometheus + Grafana | [ ] 各服务暴露 Micrometer 指标；[ ] 配 Grafana 仪表盘（JVM/MySQL/Redis/Nginx）；[ ] 配告警规则 |
| 链路 | SkyWalking | [ ] 部署 SkyWalking OAP + UI；[ ] 各服务接入 agent；[ ] 验证跨服务调用链 |

**验收标准**：Grafana 能看到全链路指标；SkyWalking 能追踪一次跨服务调用；Loki 能按 traceId 查日志。

### P2-C 高可用中间件

- [ ] **消息队列 RocketMQ**：成绩发布异步通知、考试结束自动阅卷解耦
- [ ] **定时任务 XXL-JOB**：替换 `@Scheduled`（`ExamEndTask` / `NotificationCleanupTask` 改分布式调度）
- [ ] **Redis 高可用**：主从 + 哨兵（Sentinel），热点数据（题库/试卷）多级缓存（Caffeine + Redis）
- [ ] **MySQL 高可用**：主从复制 + 读写分离（ShardingSphere 读 写分离）

**验收标准**：RocketMQ 解耦成绩通知；XXL-JOB 控制台可手动触发/查看考试结束任务；Redis 哨兵自动故障转移。

## 执行建议

1. **按 P1 → P2 顺序推进**：每个阶段是下个阶段的基础，跳级会埋坑。
2. **每个任务独立提交**：方便回滚，也让 git 历史成为学习记录。
3. **边做边写文档**：每个新技术的引入，在 `doc/` 下记录踩坑笔记（本身就是作品集加分项）。
4. **P2 是长期目标**：不必追求一次到位，可按服务逐个拆，每拆一个就上线验证。
