## P0-3 API 文档化（Knife4j）

### 改动点

**① `exam-system/pom.xml` 新增依赖**

```xml
<!-- Knife4j（基于 OpenAPI 3，比原生 Swagger UI 美观，国内主流） -->
<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    <version>4.5.0</version>
</dependency>
```

> ⚠ 注意 artifactId：
> - Spring Boot 2.7（当前）用 `knife4j-openapi3-spring-boot-starter`
> - Spring Boot 3.x（P1 升级后）改用 `knife4j-openapi3-jakarta-spring-boot-starter`（包名 `javax`→`jakarta`）
> 升级 Spring Boot 3 时记得同步切换 artifactId。

**② 新增 `exam-system/src/main/java/com/exam/config/Knife4jConfig.java`**

```java
package com.exam.config;

import io.swagger.v3.oas.models.OpenAPI;
import io.swagger.v3.oas.models.info.Info;
import io.swagger.v3.oas.models.info.License;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Profile;

@Configuration
@Profile("!prod") // 生产环境不暴露文档
public class Knife4jConfig {

    @Bean
    public OpenAPI examOpenAPI() {
        return new OpenAPI()
                .info(new Info()
                        .title("在线考试系统 API")
                        .description("题库管理 · 智能组卷 · 在线考试 · 自动阅卷 · 成绩分析")
                        .version("1.0.0")
                        .license(new License().name("MIT")));
    }
}
```

**③ `SecurityConfig.java` 放行文档路径**

在现有白名单（`/api/auth/login` 等）基础上追加：

```java
.requestMatchers(
    "/doc.html",
    "/swagger-resources/**",
    "/webjars/**",
    "/v3/api-docs/**",
    "/favicon.ico"
).permitAll()
```

**④ 为 Controller 补注解（示例）**

```java
@Tag(name = "考试管理", description = "考试的发布、查询、作答")
@RestController
@RequestMapping("/api/exam")
public class ExamController {

    @Operation(summary = "学生提交考试")
    @PostMapping("/submit")
    public Result<Void> submit(@Valid @RequestBody ExamSubmitRequest request) { ... }
}
```

### 验收

访问 `http://localhost:8081/doc.html`，看到分组清晰的在线 API 文档，可在线调试。

## P0-4 单元测试 + 覆盖率（Jacoco）

### 改动点

**① `exam-system/pom.xml` 新增 Jacoco 插件**（`<build><plugins>` 内）

```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.12</version>
    <executions>
        <execution>
            <goals><goal>prepare-agent</goal></goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals><goal>report</goal></goals>
        </execution>
        <execution>
            <id>jacoco-check</id>
            <goals><goal>check</goal></goals>
            <configuration>
                <rules>
                    <rule>
                        <element>BUNDLE</element>
                        <limits>
                            <limit>
                                <counter>LINE</counter>
                                <value>COVEREDRATIO</value>
                                <minimum>0.40</minimum> <!-- 初期门槛 40%，逐步提高 -->
                            </limit>
                        </limits>
                    </rule>
                </rules>
            </configuration>
        </execution>
    </executions>
</plugin>
```

> 说明：`spring-boot-starter-test` 已含 JUnit5 + Mockito + AssertJ，无需额外引入。
> 项目 `src/test/java/com/exam/perf/` 下已有压测造数据工具，但**无业务单元测试**——这是要补的。

**③ 重点测试目标**（按业务复杂度排序）

| 优先级 | 测试对象 | 测试场景 |
|:---:|:---|:---|

## P0-5 代码规范（Spotless）

### 改动点

**① `exam-system/pom.xml` 新增 Spotless 插件**

```xml
<plugin>
    <groupId>com.diffplug.spotless</groupId>
    <artifactId>spotless-maven-plugin</artifactId>
    <version>2.43.0</version>
    <configuration>
        <java>
            <includes>
                <include>src/main/java/**/*.java</include>
                <include>src/test/java/**/*.java</include>
            </includes>
            <googleJavaFormat>
                <version>1.17.0</version>
                <style>AOSP</style> <!-- 4 空格缩进，更接近现有代码风格 -->
            </googleJavaFormat>
            <removeUnusedImports/>
            <trimTrailingWhitespace/>
            <endWithNewline/>
        </java>
    </configuration>
    <executions>
        <execution>
            <phase>verify</phase>
            <goals><goal>check</goal></goals>
        </execution>
    </executions>
</plugin>
```

**② 项目根新增 `.editorconfig`**

```ini
root = true

[*.{java,xml}]
```

## P0-6 CI 流水线

### 改动点

**新增 `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '11'
          cache: maven
      - run: mvn -B verify -f exam-system/pom.xml
      - run: mvn -B spotless:check -f exam-system/pom.xml

        with:
          name: jacoco-report
          path: exam-system/target/site/jacoco/

  frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: npm
          cache-dependency-path: art-design-pro-ui/package-lock.json
      - run: npm ci
        working-directory: art-design-pro-ui
      - run: npm run lint
        working-directory: art-design-pro-ui

        working-directory: art-design-pro-ui

    runs-on: ubuntu-latest
```

## P1-1 Spring Boot 3 + Java 17 升级（重点技术债）

> 这是最复杂的改动，涉及多处 API 变更。建议先在独立分支操作。

### 改动点

**① `exam-system/pom.xml`**

```xml
<!-- Java 版本 -->
<java.version>17</java.version>

<!-- Spring Boot parent -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.12</version> <!-- 或最新 3.2.x -->
</parent>

<!-- jjwt 升级（API 有 breaking change） -->
<jjwt.version>0.12.6</jjwt.version>

<!-- 移除 Lettuce 覆盖（Spring Boot 3 自带新版，注释里也提到了） -->
<!-- 删除 <lettuce.version>6.5.1.RELEASE</lettuce.version> 这一行 -->
```

**② `javax.*` → `jakarta.*`（用 OpenRewrite 自动化）**

```bash
# 在 exam-system 目录
mvn org.openrewrite.maven:rewrite-maven-plugin:run \
  -Drewrite.activeRecipes=org.openrewrite.java.migrate.JavaxToJakarta
```

或手动改（项目里 `javax` 用得少，主要这几处）：

| 当前 import | 改为 |
|:---|:---|
| `javax.annotation.PostConstruct` | `jakarta.annotation.PostConstruct` |
| `javax.servlet.*` | `jakarta.servlet.*` |
| `javax.validation.*` | `jakarta.validation.*` |
| `javax.persistence.*` | `jakarta.persistence.*` |

> 排查命令：`grep -rn "javax\." exam-system/src/main/java/`

**③ `JwtUtils.java` 重写（jjwt 0.12 API 变更）**

当前代码（0.11.x，升级后全部失效）：
```java
// ❌ 0.12 移除的 API
Jwts.builder().setSubject(...)          // → .subject(...)
           .setIssuedAt(...)            // → .issuedAt(...)
           .setExpiration(...)          // → .expiration(...)
           .signWith(key, SignatureAlgorithm.HS256);  // → .signWith(key, Jwts.SIG.HS256)
Jwts.parserBuilder().setSigningKey(key).build()       // → Jwts.parser().verifyWith(key).build()
.parseClaimsJws(token).getBody()                      // → .parseSignedClaims(token).getPayload()
```

改为（0.12.x）：
```java
public String generateToken(Long userId, String username, String roleCode) {
    return Jwts.builder()
            .subject(String.valueOf(userId))
            .claim("username", username)
            .claim("roleCode", roleCode)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(key, Jwts.SIG.HS256)
            .compact();
}

public Claims parseToken(String token) {
    return Jwts.parser()
            .verifyWith(key)
            .build()
            .parseSignedClaims(token)
            .getPayload();
}
```

**④ Knife4j artifactId 切换**（若 P0-3 已做）

```xml
<!-- 从 -->
<artifactId>knife4j-openapi3-spring-boot-starter</artifactId>
<!-- 改为 -->
<artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
```

**⑤ 验证依赖兼容性**

| 依赖 | 当前 | Spring Boot 3.2 兼容性 | 处理 |
|:---|:---|:---|:---|
| MyBatis-Plus | 3.5.5 | ✅ ≥3.5.5 支持 | 保持或升 3.5.7 |
| EasyExcel | 3.3.3 | ✅ 兼容 | 保持 |
| Hutool | 5.8.25 | ✅ 兼容 | 保持或升 5.8.27 |
| Guava | 33.0.0 | ✅ 兼容 | 保持 |
| jjwt | 0.11.5 | ❌ 需升 0.12.6 | 升级 + 改 API |
| Lettuce | 6.5.1（覆盖） | ✅ Boot 3 自带新版 | **移除覆盖** |

### 验收

`mvn spring-boot:run` 在 Java 17 下正常启动；三端功能回归通过；`mvn verify`（含测试）通过。
