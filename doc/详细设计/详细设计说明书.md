## 三、认证模块详细设计

**涉及类**：AuthController、AuthService、JwtUtils、SecurityConfig、JwtAuthenticationFilter、LoginRateLimiter、TokenBlacklistService

### 3.1 登录流程 `POST /api/auth/login`

**输入**：`{ username, password, captchaKey(可选), captchaCode(可选) }`

```
1. 验证码校验 → 前端用ArtDragVerify拖拽滑块替代，后端可选图形验证码
2. 频率检查(LoginRateLimiter) → 5次失败锁定15分钟
3. 校验用户名存在 → 不存在记录失败返回400
4. BCrypt密码匹配 → 不匹配记录失败返回400
5. 校验status=1 → 0则返回403"账号已被禁用"
6. 生成JWT(sub:userId, username, roleCode, exp:24h)
7. 清除失败计数 → 返回Token+userInfo
```

### 3.2 JWT过滤器 `JwtAuthenticationFilter`

```
每个请求 → 提取Authorization头
├── 无 → 跳过（匿名）
└── 有 → 提取Bearer Token → 检查黑名单
    ├── 在黑名单 → 跳过
    └── 不在 → parseToken
        ├── 成功 → 构建Authentication → SecurityContext
        └── 失败 → 跳过（后续返回401）
```

### 3.3 登出 `POST /api/auth/logout`

Token加入内存黑名单`ConcurrentHashMap<Token,过期时间>`，定期清理过期条目。

### 3.4 Security配置

```
csrf.disable + STATELESS + frameOptions.sameOrigin
permitAll: /api/auth/login, /api/auth/captcha, /api/auth/logout, /ws/**, /uploads/**
其余: authenticated
异常: 401→未登录, 403→无权限
JwtFilter在UsernamePasswordAuthenticationFilter之前
```

## 七、试卷管理模块 (TEACHER)

### 7.1 手动组卷 `POST /api/paper/add`

校验题目存在且同学科 → 无重复 → totalScore=SUM(score) → insert Paper+PaperQuestion

### 7.2 随机组卷 `POST /api/paper/random`

```

  查询candidates(subject+creator+type+difficulty)
  → 不足则报错"题库不足"
  → shuffle + subList(0, count) → 设分值
totalScore = SUM(count×scorePerQuestion)
```

读取模板规则 → 按随机组卷逻辑生成 → 返回新试卷ID

## 九、在线考试模块详细设计 (STUDENT)

**涉及类**：StudentExamController、StudentExamService、ExamRecordMapper、ExamAnswerMapper

### 9.1 开始考试 `GET /api/student/exam/start/{examId}`

```
1. 考试状态：NOW() BETWEEN start_time AND end_time → 否则拒绝
2. 班级权限：学生class_id == exam.class_id → 否则拒绝
3. 考试记录检查：
   ├── 无记录 → INSERT exam_record(status=1, start_time=NOW())
   ├── status=1 → 返回已有记录（断点续答）
   └── status>=2 → "您已完成该考试"
4. 获取试卷题目（不返回answer和analysis，防作弊）
5. 防作弊：
   ├── 试题乱序：shuffleWithSeed(questions, recordId) 稳定随机
   └── 选项乱序：relabelOptions 乱序后重标记ABCD
6. 返回：recordId, examName, duration, endTime, serverTime, questions[]
```

### 9.2 自动保存 `POST /api/student/exam/auto-save`

前端每30秒调用，对每个答案执行UPSERT（存在则UPDATE，不存在则INSERT）。

### 9.3 提交试卷 `POST /api/student/exam/submit`

```
1. 身份校验(record.userId==currentUser) + 状态校验(status==1)
2. 超时校验：NOW() <= exam.endTime + 60秒宽限期
3. UPSERT保存所有答案
4. 执行自动判分（见第十章）
5. 更新exam_record：
   ├── 有简答题 → status=2(待批改), total_score=null
   └── 纯客观题 → status=3(已批改), total_score=objective_score
6. WebSocket通知教师
```

### 9.4 切屏记录 `POST /api/student/exam/switch-screen`

原子递增switch_count → 返回当前次数 → 前端判断超限则自动交卷

### 9.5 倒计时机制

```
前端：remainSeconds=(endTime-serverTime)/1000 → setInterval每秒递减
  ≤300秒→提醒 → ≤60秒→红色脉冲 → =0→自动提交
后端兜底：提交时NOW()<=endTime+60秒
```

## 十、自动判分算法详细设计

### 10.1 总体流程

```
提交试卷 → 遍历答题记录
├── type=1(单选) → 精确匹配
├── type=2(多选) → 集合比较（漏选半分）
├── type=3(判断) → 精确匹配
├── type=4(填空) → 逐空匹配
└── type=5(简答) → score=0, isCorrect=NULL（待批改）
→ 累加objectiveScore
```

### 10.2 单选题/判断题 (type=1,3)

```
studentAnswer.trim().equalsIgnoreCase(correctAnswer.trim())
→ 匹配: score=fullScore, isCorrect=1
→ 不匹配: score=0, isCorrect=0
```

### 10.3 多选题 (type=2)

```
correctSet = correctAnswer.split(",")  // {A,B,C}
studentSet = studentAnswer.split(",")  // {A,B}

完全匹配         → score=fullScore, isCorrect=1
correctSet包含studentSet → score=fullScore×0.5, isCorrect=2（漏选半分）
其他(错选/多选)   → score=0, isCorrect=0
```

### 10.4 填空题 (type=4)

```
correctParts = correctAnswer.split(",")  // ["Java","JVM"]
studentParts = studentAnswer.split(",")  // ["java","JVM"]

逐空ignoreCase比较 → correctCount
score = fullScore × (correctCount / totalBlanks)
isCorrect = 全对→1 / 部分对→2 / 全错→0
```

### 10.5 判分结果标记

| is_correct | 含义 | 错题本收集 |
|:----------:|------|:----------:|
| 0 | 错误 | ✅ |
| 1 | 正确 | ❌ |
| 2 | 部分正确 | 视实现 |
| NULL | 待批改 | 批改后判定 |

## 十一、阅卷管理模块 (TEACHER)

### 11.1 待批改列表 `GET /api/marking/list/{examId}`

查询exam_record WHERE status=2，关联用户和班级信息

### 11.2 批改打分 `POST /api/marking/score`

```json
{ "recordId": 1, "scores": [
    { "answerId": 125, "score": 8.00, "comment": "回答基本正确" }
]}
```

```
1. 校验record.status=2，score>=0且<=满分
2. 更新exam_answer: score, is_correct(满分→1/0分→0/中间→2), comment
3. subjectiveScore = SUM(简答题score)
4. 更新exam_record:
   subjective_score, total_score=objective+subjective, status=3
```

### 11.3 发布成绩 `POST /api/marking/publish/{examId}`

```
检查全部批改完(status=2的count==0) → UPDATE exam_exam SET score_published=1
→ WebSocket通知学生"成绩已发布"
```

## 十三、错题本模块 (STUDENT)

### 13.1 收集机制

自动判分后is_correct=0的记录自动成为错题，查询时过滤is_removed=0

### 13.2 错题科目 `GET /api/wrong/subjects`

```sql
SELECT subject_id, subject_name, COUNT(*) AS wrongCount
FROM exam_answer a JOIN exam_record r JOIN exam_question q JOIN edu_subject s
WHERE r.user_id=? AND a.is_correct=0 AND a.is_removed=0
GROUP BY subject_id
```

### 13.3 移除错题 `DELETE /api/wrong/{answerId}`

校验所有权(record.user_id) → 软删除：is_removed=1
