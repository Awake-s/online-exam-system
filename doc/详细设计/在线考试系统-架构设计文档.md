## 二、后端架构详细设计

### 2.1 整体采用经典三层架构（MVC）

```
Controller层（表现层）
    ↓ 调用
Service层（业务逻辑层）
    ↓ 调用
Mapper层（数据访问层）
    ↓ 操作
MySQL数据库
```

这是Spring Boot项目最标准的分层方式，每一层职责清晰：
- Controller：接收HTTP请求、参数校验、调用Service、返回响应
- Service：处理业务逻辑、事务管理、数据组装
- Mapper：与数据库交互，执行SQL

### 2.3 各层代码职责与调用关系

以"教师发布考试"为例，展示一个完整的请求链路：

```
前端点击"发布" → POST /api/exam/add {examName, paperId, classId, startTime, endTime}
    │
    ▼
ExamController.addExam(@RequestBody @Valid ExamPublishRequest request)
    │  @PreAuthorize("hasRole('TEACHER')")  ← 权限校验
    │  @Valid ← 参数校验（JSR303注解）
    ▼
ExamService.publishExam(request, currentUserId)
    │  1. 校验试卷是否存在且属于当前教师
    │  2. 校验试卷状态是否为"已发布"
    │  3. 校验班级是否存在
    │  4. 校验时间合法性（startTime < endTime，startTime > now）
    │  5. 校验该班级该时间段无重叠考试
    │  6. 构建ExamExam实体，设置creator_id
    │  @Transactional ← 事务管理
    ▼
ExamExamMapper.insert(examExam)
    │  MyBatis-Plus自动生成INSERT语句
    ▼
MySQL: INSERT INTO exam_exam (exam_name, paper_id, class_id, ...) VALUES (...)
    │
    ▼
返回 Result.success("发布成功")
```

```java
@RestController
@RequestMapping("/api/question")
@PreAuthorize("hasRole('TEACHER')")
public class QuestionController {

    @Autowired
    private QuestionService questionService;

    @GetMapping("/list")
    public Result<PageResult<QuestionResponse>> list(
            @RequestParam(defaultValue = "1") Integer page,
            @RequestParam(defaultValue = "10") Integer size,
            @RequestParam(required = false) Long subjectId,
            @RequestParam(required = false) Integer questionType,
            @RequestParam(required = false) Integer difficulty,
            @RequestParam(required = false) String keyword) {
        Long currentUserId = SecurityUtils.getCurrentUserId();
        return Result.success(questionService.listQuestions(
            page, size, subjectId, questionType, difficulty, keyword, currentUserId));
    }

    @PostMapping("/add")
    public Result<Void> add(@RequestBody @Valid QuestionAddRequest request) {
        Long currentUserId = SecurityUtils.getCurrentUserId();
        questionService.addQuestion(request, currentUserId);
        return Result.success(null);
    }

    @DeleteMapping("/{id}")
    public Result<Void> delete(@PathVariable Long id) {
        Long currentUserId = SecurityUtils.getCurrentUserId();
        questionService.deleteQuestion(id, currentUserId);
        return Result.success(null);
    }
}
```

**Service示例 QuestionServiceImpl.java：**
```java
@Service
public class QuestionServiceImpl implements QuestionService {

    @Autowired
    private ExamQuestionMapper questionMapper;
    @Autowired
    private ExamPaperQuestionMapper paperQuestionMapper;

    @Override
    public PageResult<QuestionResponse> listQuestions(
            Integer page, Integer size, Long subjectId,
            Integer questionType, Integer difficulty, String keyword, Long creatorId) {
        // 构建查询条件（数据隔离：只查自己创建的）
        LambdaQueryWrapper<ExamQuestion> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(ExamQuestion::getCreatorId, creatorId);
        wrapper.eq(subjectId != null, ExamQuestion::getSubjectId, subjectId);
        wrapper.eq(questionType != null, ExamQuestion::getQuestionType, questionType);
        wrapper.eq(difficulty != null, ExamQuestion::getDifficulty, difficulty);
        wrapper.like(keyword != null, ExamQuestion::getContent, keyword);
        wrapper.orderByDesc(ExamQuestion::getCreateTime);

        // MyBatis-Plus分页查询
        Page<ExamQuestion> pageResult = questionMapper.selectPage(
            new Page<>(page, size), wrapper);

        // 转换为响应DTO
        // ...
    }

    @Override
    public void deleteQuestion(Long id, Long creatorId) {
        // 1. 检查是否是自己创建的
        ExamQuestion question = questionMapper.selectById(id);
        if (question == null) throw new NotFoundException("题目不存在");
        if (!question.getCreatorId().equals(creatorId)) throw new BusinessException("无权删除");

        // 2. 检查是否被试卷引用
        Long count = paperQuestionMapper.selectCount(
            new LambdaQueryWrapper<ExamPaperQuestion>()
                .eq(ExamPaperQuestion::getQuestionId, id));
        if (count > 0) throw new BusinessException("该题目已被试卷使用，无法删除");

        // 3. 执行删除
        questionMapper.deleteById(id);
    }
}
```

```xml
        AVG(total_score) as avgScore,
        MAX(total_score) as maxScore,
        MIN(total_score) as minScore,
        SUM(CASE WHEN total_score >= #{passScore} THEN 1 ELSE 0 END) as passCount
    FROM exam_record
    WHERE exam_id = #{examId} AND status = 3
</select>
```

### 2.5 Security安全架构

```
HTTP请求进入
    │
    ▼
JwtAuthenticationFilter（自定义过滤器，在UsernamePasswordFilter之前）
    │
    ├── 请求Header有Token → 解析Token → 校验有效期
    │       │
    │       ├── 有效 → 从Token取出userId和roleCode → 构建Authentication → 放入SecurityContext
    │       │
    │       └── 无效/过期 → 不设置Authentication（后续由权限注解拦截）
    │
    └── 请求Header无Token → 跳过（匿名访问）
    │
    ▼
Spring Security权限判断
    │
    ├── /api/auth/login, /api/auth/captcha → permitAll（放行）
    │
    ├── @PreAuthorize("hasRole('ADMIN')") → 检查SecurityContext中的角色
    │       │
    │       ├── 角色匹配 → 放行到Controller
    │       └── 角色不匹配 → 返回403
    │
    └── 未认证（无Token）→ 返回401
```

**SecurityConfig.java 核心配置：**
```java
@Configuration
@EnableMethodSecurity  // 启用方法级权限注解
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()                    // 前后端分离不需要CSRF
            .sessionManagement()
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS)  // 无状态，用JWT
            .and()
            .authorizeRequests()
                .antMatchers("/api/auth/login", "/api/auth/captcha").permitAll()
                .antMatchers("/uploads/**").permitAll()  // 静态资源放行
                .anyRequest().authenticated()
            .and()
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();  // 密码加密方式
    }
}
```

**Axios封装 request.js：**
```javascript
import axios from 'axios'
import { getToken, removeToken } from '@/utils/auth'
import router from '@/router'
import { ElMessage } from 'element-plus'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
})

// 请求拦截器：自动携带Token
request.interceptors.request.use(config => {
  const token = getToken()
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器：统一错误处理
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code !== 200) {
      ElMessage.error(res.message || '操作失败')
      return Promise.reject(res)
    }
    return res
  },
  error => {
    if (error.response?.status === 401) {
      removeToken()
      router.push('/login')
      ElMessage.error('登录已过期，请重新登录')
    } else if (error.response?.status === 403) {
      ElMessage.error('无权限访问')
    } else {
      ElMessage.error('网络错误')
    }
    return Promise.reject(error)
  }
)

export default request
```

```javascript
      this.token = res.data.token

      this.token = ''
      this.userInfo = {}
      removeToken()
    }
  }
})
```

### 4.2 MyBatis-Plus使用策略

| 场景 | 方式 | 说明 |
|------|------|------|
| 单表CRUD | BaseMapper自带方法 | selectById, insert, updateById, deleteById |
| 条件查询 | LambdaQueryWrapper | 链式构建WHERE条件，类型安全 |
| 分页查询 | Page + selectPage | 配合分页插件自动处理LIMIT |
| 关联查询 | Mapper XML | 多表JOIN、统计查询、复杂SQL |
| 批量操作 | saveBatch / XML | 批量插入答题记录等 |

```xml
<select id="getWrongList" resultType="WrongDetailResponse">
    SELECT
        a.id as answerId,
        a.question_id as questionId,
        q.question_type as questionType,
        q.content,
        a.answer as myAnswer,
        q.answer as correctAnswer,
        q.analysis,
        s.subject_name as subjectName,
        e.exam_name as examName
    FROM exam_answer a
    JOIN exam_record r ON a.record_id = r.id
    JOIN exam_question q ON a.question_id = q.id
    JOIN edu_subject s ON q.subject_id = s.id
    JOIN exam_exam e ON r.exam_id = e.id
    WHERE r.user_id = #{userId}
      AND a.is_correct = 0
      AND a.is_removed = 0
      <if test="subjectId != null">
          AND q.subject_id = #{subjectId}
      </if>
```

### 7.3 生产部署架构

```
                    ┌──────────────┐
                    │    Nginx     │
                    │   端口：80    │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
        静态资源        /api/*       /uploads/*
     (Vue打包产物)    (反向代理)    (静态文件)
     dist/index.html     │
                         ▼
                  ┌──────────────┐
                  │ Spring Boot  │
                  │  端口：8080   │
                  └──────┬───────┘
                         │
                         ▼
                  ┌──────────────┐
                  │   MySQL 8.0  │
                  │  端口：3306   │
                  └──────────────┘
```

**Nginx配置：**
```nginx
server {
    listen 80;
    server_name localhost;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;  # Vue打包后的dist目录
        try_files $uri $uri/ /index.html;  # SPA路由支持
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:8080/api/;
    }

    # 上传文件访问
    location /uploads/ {
        alias /path/to/uploads/;
    }
}
```
