## 一、权限控制详细设计

### 1.1 认证流程

```
用户登录 → 验证用户名+密码(BCrypt) → 生成JWT Token(含userId, roleCode)
    → 前端存储Token到localStorage → 每次请求Header携带 Authorization: Bearer <token>
    → 后端拦截器解析Token → 校验有效期 → 注入SecurityContext
```

**JWT Token 结构：**
```json
{
  "sub": "1",              // 用户ID
  "username": "admin",     // 用户名
  "roleCode": "ADMIN",     // 角色编码
  "iat": 1706000000,       // 签发时间
  "exp": 1706086400        // 过期时间（24小时）
}
```

**Token 过期策略：**
- Token有效期：24小时
- Token过期后端返回401，前端响应拦截器捕获401状态码（带3秒防抖），自动清除本地Token并跳转登录页
- 用户需重新登录获取新Token
- 登出时后端将Token加入内存黑名单（`TokenBlacklistService`），定时清理过期条目

### 1.2 登录验证机制

**前端拖拽滑块验证：**
```
前端加载登录页 → 显示拖拽滑块验证组件（ArtDragVerify）
    → 用户拖动滑块到指定位置完成验证 → isPassing 置为 true
    → 点击登录时检查 isPassing → 通过后发送登录请求 → 后端校验用户名+密码
    → 登录成功/失败后自动重置滑块状态
```

**验证机制说明：**
- 前端采用 `ArtDragVerify` 拖拽滑块作为人机验证，验证通过后方可提交登录表单
- 后端同时提供 `/api/auth/captcha` 图形验证码接口（Hutool CaptchaUtil 实现），登录时后端会校验验证码参数（captchaKey + captchaCode），若前端未传则跳过校验
- 当前前端使用拖拽滑块替代图形验证码，图形验证码作为可选增强机制保留
- 登录失败次数由后端 `LoginRateLimiter` 控制，5次失败后锁定账户15分钟

## 二、自动判分算法详细设计

### 2.1 判分总体流程

```
学生提交试卷
    │
    ▼
遍历每道答题记录
    │
    ├── 题型=1(单选) → 精确匹配判分
    ├── 题型=2(多选) → 多选判分（支持漏选半分）
    ├── 题型=3(判断) → 精确匹配判分
    ├── 题型=4(填空) → 填空匹配判分
    └── 题型=5(简答) → 标记为待批改，得分暂为0
    │
    ▼
计算客观题总分 → 更新exam_record.objective_score
更新exam_record.status = 2(已交卷)
```

### 2.2 各题型判分规则

**单选题（question_type = 1）：**
```
规则：学生答案与正确答案完全一致则得满分，否则0分
示例：正确答案="A"，学生答案="A" → 满分
      正确答案="A"，学生答案="B" → 0分
```

**多选题（question_type = 2）：**
```
规则：
  - 完全正确：得满分
  - 漏选（选的都对但不全）：得该题分值的50%
  - 多选或错选：0分

示例：正确答案="A,B,C"
  学生答案="A,B,C" → 满分
  学生答案="A,B"   → 50%分值（漏选，is_correct=2部分正确）
  学生答案="A,B,D" → 0分（错选）
  学生答案="A"     → 50%分值（漏选）

判分伪代码：
  correctSet = 正确答案.split(",") → Set
  studentSet = 学生答案.split(",") → Set
  if (studentSet == correctSet) → 满分
  else if (correctSet.containsAll(studentSet)) → 50%分值
  else → 0分
```

**判断题（question_type = 3）：**
```
规则：学生答案与正确答案完全一致则得满分，否则0分
正确答案格式："1"表示正确，"0"表示错误
示例：正确答案="1"，学生答案="1" → 满分
```

**填空题（question_type = 4）：**
```
规则：
  - 单个空：去除首尾空格后，忽略大小写比较
  - 多个空：答案用英文逗号分隔，逐个比较
  - 每个空独立计分，总分 = 该题分值 × (正确空数 / 总空数)

示例：正确答案="Java,JVM"（2个空）
  学生答案="java,JVM"  → 满分（忽略大小写）
  学生答案="Java,jdk"  → 50%分值（1个对1个错）
  学生答案="java"      → 50%分值（只填了1个空）

判分伪代码：
  correctParts = 正确答案.split(",")
  studentParts = 学生答案.split(",")
  correctCount = 0
  for i in range(correctParts.length):
      if i < studentParts.length:
          if correctParts[i].trim().equalsIgnoreCase(studentParts[i].trim()):
              correctCount++
  score = 题目分值 × (correctCount / correctParts.length)
```

**简答题（question_type = 5）：**
```
规则：不自动判分，标记为待批改
  - is_correct = NULL
  - score = 0（待教师手动打分）
  - exam_record.status = 2（已交卷，非已批改）
```

### 2.3 判分结果标记

| is_correct值 | 含义 | 适用场景 |
|:---:|------|----------|
| 0 | 错误 | 客观题答错 |
| 1 | 正确 | 客观题答对（满分） |
| 2 | 部分正确 | 多选漏选、填空部分正确 |
| NULL | 待批改 | 简答题未批改 |

## 三、考试时间控制详细设计

### 3.1 考试状态流转

```
考试创建(status=0 未开始)
    │
    │  到达start_time
    ▼
进行中(status=1)
    │
    │  到达end_time
    ▼
已结束(status=2)
```

**状态判断策略：查询时实时计算（不用定时任务）**
```sql
-- 查询考试列表时动态计算状态
SELECT *,
  CASE
    WHEN NOW() < start_time THEN 0  -- 未开始
    WHEN NOW() BETWEEN start_time AND end_time THEN 1  -- 进行中
    ELSE 2  -- 已结束
  END AS status
FROM exam_exam
```

> 选择查询时计算而非定时任务的原因：实现简单、无延迟、不依赖额外组件。

### 3.2 考试倒计时机制

**前端倒计时：**
```
1. 开始考试时，后端返回服务器当前时间和考试结束时间
2. 前端计算剩余秒数 = (end_time - 服务器时间)
3. 前端启动setInterval每秒倒计时
4. 剩余5分钟时弹出提醒
5. 倒计时归零 → 自动调用提交接口
```

**后端兜底校验：**
```
提交试卷时后端校验：
  if (NOW() > exam.end_time + 60秒宽限期):
      拒绝提交，返回"考试已结束"
  else:
      正常接收提交（允许60秒网络延迟宽限）
```

### 3.3 断网/异常处理

| 异常场景 | 处理方案 |
|----------|----------|
| 学生中途关闭浏览器 | exam_record保持status=1(答题中)，重新打开可继续答题 |
| 学生中途断网 | 前端答案暂存localStorage，恢复网络后可继续 |
| 考试时间到但学生未交卷 | 前端倒计时归零自动提交；若前端未触发，后端 `ExamEndTask` 每60秒扫描并自动交卷+批改客观题 |
| 学生重复点击"开始考试" | 后端检查exam_record是否已存在，存在则返回已有记录继续答题 |

### 3.4 防作弊基础措施

| 措施 | 实现方式 |
|------|----------|
| 禁止重复考试 | exam_record表(exam_id, user_id)唯一约束 |
| 试题乱序 | 后端使用 recordId 作为稳定随机种子，按题型分组后组内乱序（`shuffleWithSeed`），通过 antiCheatConfig 配置开关 |
| 选项乱序 | 后端对选择题选项乱序后重新标记为顺序ABCD（`relabelOptions`），通过 antiCheatConfig 配置开关 |
| 切屏检测 | 前端监听visibilitychange+blur事件，后端 `recordSwitchScreen` 原子递增计数，超过最大切屏次数强制交卷 |

## 四、接口详细设计（请求/响应数据结构）

### 4.0 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

**错误码定义：**

| 错误码 | 含义 | 说明 |
|:------:|------|------|
| 200 | 成功 | 操作成功 |
| 400 | 参数错误 | 请求参数校验失败 |
| 401 | 未认证 | Token无效或过期 |
| 403 | 无权限 | 角色权限不足 |
| 404 | 资源不存在 | 请求的数据不存在 |
| 409 | 业务冲突 | 如重复考试、删除被引用数据等 |
| 500 | 服务器错误 | 系统内部异常 |

**分页请求通用参数：**
```
page: 页码（从1开始，默认1）
size: 每页条数（默认10，最大100）
```

**分页响应通用格式：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 10,
    "pages": 10
  }
}
```

## 五、异常处理与边界情况详细设计

### 5.1 全局异常处理

```java
// 统一异常处理器 @RestControllerAdvice
GlobalExceptionHandler {

    // 参数校验异常 → 400
    handleValidationException(MethodArgumentNotValidException e)
        → { code: 400, message: "参数错误：用户名不能为空" }

    // 业务异常 → 409
    handleBusinessException(BusinessException e)
        → { code: 409, message: e.getMessage() }

    // 权限不足 → 403
    handleAccessDeniedException(AccessDeniedException e)
        → { code: 403, message: "无权限访问" }

    // 认证失败 → 401
    handleAuthenticationException(AuthenticationException e)
        → { code: 401, message: "未登录或Token已过期" }

    // 资源不存在 → 404
    handleNotFoundException(NotFoundException e)
        → { code: 404, message: "资源不存在" }

    // 其他异常 → 500
    handleException(Exception e)
        → { code: 500, message: "服务器内部错误" }
        → 记录错误日志
}
```

### 5.2 删除操作关联检查汇总

| 删除对象 | 检查关联 | 有关联时的处理 |
|----------|----------|----------------|
| 删除用户（教师） | exam_question.creator_id, exam_paper.creator_id, exam_exam.creator_id | 返回409："该教师有关联数据，无法删除" |
| 删除用户（学生） | exam_record.user_id | 返回409："该学生有考试记录，无法删除" |
| 删除班级 | sys_user.class_id, exam_exam.class_id | 返回409："该班级下有学生/考试，无法删除" |
| 删除科目 | exam_question.subject_id | 返回409："该科目下有题目，无法删除" |
| 删除题目 | exam_paper_question.question_id | 返回409："该题目已被试卷使用，无法删除" |
| 删除试卷 | exam_exam.paper_id | 返回409："该试卷已被考试使用，无法删除" |
| 删除考试 | exam_record（status>=1） | 返回409："已有学生参加考试，无法删除" |

### 5.3 考试相关边界情况

| 场景 | 处理方案 |
|------|----------|
| 学生在考试结束前1秒点击开始 | 允许开始，但实际答题时间只有1秒+60秒宽限 |
| 学生提交时网络超时 | 前端重试机制（最多3次），后端幂等处理（已提交则返回成功） |
| 教师在考试进行中修改试卷 | 禁止：试卷被考试引用后不可编辑 |
| 教师在考试进行中删除考试 | 禁止：只能删除未开始的考试 |
| 同一学生同时在两个浏览器打开考试 | 通过exam_record唯一约束保证只有一条记录，两个浏览器操作同一条记录 |
| 批改时给分超过满分 | 后端校验 score <= fullScore，超过则返回400 |
| 试卷没有主观题 | 提交后直接status=3(已批改)，total_score=objective_score |

### 5.4 并发控制

| 场景 | 方案 |
|------|------|
| 两个管理员同时编辑同一用户 | 乐观锁：基于update_time字段，更新时检查版本 |
| 学生重复提交试卷 | 数据库唯一约束 + 后端状态检查（status必须为1才能提交） |
| 教师同时批改同一份试卷 | 实际场景不存在（一份试卷只有一个教师批改），无需特殊处理 |

### 8.2 自动判分核心方法设计

```java
/**
 * 自动判分方法（在StudentExamService.submitExam中调用）
 * 
 * @param answers    学生答案列表
 * @param questions  试卷题目列表（含正确答案和分值）
 * @return 客观题总分
 */
BigDecimal autoGrade(List<ExamAnswer> answers, List<PaperQuestion> questions) {
    BigDecimal objectiveScore = BigDecimal.ZERO;
    
    for (ExamAnswer answer : answers) {
        PaperQuestion pq = findQuestion(questions, answer.getQuestionId());
        int type = pq.getQuestionType();
        
        if (type == 5) {
            // 简答题：跳过，待人工批改
            answer.setIsCorrect(null);
            answer.setScore(BigDecimal.ZERO);
            continue;
        }
        
        BigDecimal score;
        switch (type) {
            case 1:
            case 3:
                score = gradeSingleChoice(answer, pq);   // 单选、判断
                break;
            case 2:
                score = gradeMultiChoice(answer, pq);     // 多选
                break;
            case 4:
                score = gradeFillBlank(answer, pq);       // 填空
                break;
            default:
                score = BigDecimal.ZERO;
        }
        
        answer.setScore(score);
        objectiveScore = objectiveScore.add(score);
    }
    
    return objectiveScore;
}
```

## 十一、新增模块接口设计（v2.0补充）

### 11.1 专业管理 /api/major（管理员）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/major/list | GET | 专业分页列表（page, size, majorName） |
| /api/major/all | GET | 所有专业（下拉框用，不分页） |
| /api/major/add | POST | 添加专业（majorName, description） |
| /api/major/update | PUT | 更新专业 |
| /api/major/{id} | DELETE | 删除专业（有班级关联时禁止删除） |

### 11.2 试卷模版 /api/template（教师）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/template/list | GET | 模版列表（page, size, subjectId） |
| /api/template/{id} | GET | 模版详情（含组卷规则） |
| /api/template/add | POST | 创建模版（templateName, subjectId, rules[]） |
| /api/template/update | PUT | 更新模版 |
| /api/template/{id} | DELETE | 删除模版 |
| /api/template/generate/{id} | POST | 根据模版自动组卷生成试卷 |

### 11.3 通知管理 /api/notification（所有角色）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/notification/list | GET | 当前用户通知列表（page, size, type） |
| /api/notification/unread-count | GET | 未读通知数量 |
| /api/notification/read/{id} | PUT | 标记单条已读 |
| /api/notification/read-all | PUT | 全部标记已读 |
| /api/notification/{id} | DELETE | 删除通知 |

通知类型：EXAM_PUBLISHED（考试发布）、SCORE_PUBLISHED（成绩发布）、EXAM_SUBMITTED（学生交卷）等。
通知通过WebSocket实时推送到前端，前端顶部栏显示红点提醒。

### 11.4 即时通讯 /api/chat（所有角色）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/chat/conversations | GET | 会话列表 |
| /api/chat/conversations | POST | 创建/获取会话 |
| /api/chat/messages/{conversationId} | GET | 历史消息（分页） |

WebSocket端点：`/ws`（STOMP over SockJS）
- 订阅：`/user/queue/messages`（接收私信）
- 发送：`/app/chat.send`（发送消息）
- 连接时通过JWT认证（WebSocketAuthInterceptor）

### 11.5 文件管理 /api/file（所有角色）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/file/upload | POST | 上传文件（multipart/form-data） |
| /api/file/download/{filename} | GET | 下载文件 |

文件类型校验：仅允许图片（jpg/png/gif）、Excel（xlsx/xls）等白名单扩展名。

### 11.6 防作弊详细设计

| 措施 | 前端实现 | 触发条件 |
|------|----------|----------|
| 全屏模式 | Fullscreen API强制全屏 | 确认考试须知后进入 |
| 切屏检测 | visibilitychange + blur事件 | 超过最大切屏次数自动交卷 |
| 禁止复制粘贴 | CSS user-select: none + JS事件拦截 | 考试全程 |
| 无操作超时 | 监听鼠标/键盘事件，无操作超时自动交卷 | 考试全程 |
| 倒计时脉冲 | 最后1分钟倒计时红色脉冲动画 | remainSeconds <= 60 |
| 须知弹窗 | 考前须知 + 勾选确认 | 进入考试页面时 |

### 11.7 定时任务设计

| 任务 | 类 | 执行频率 | 说明 |
|------|-----|----------|------|
| 考试自动结束 | ExamEndTask | 每60秒（fixedDelay） | 扫描2小时内刚结束的考试→未交卷学生自动交卷并批改客观题→创建缺考记录→通知学生和教师 |
| 已读通知清理 | NotificationCleanupTask | 每天凌晨3点 | 清理30天前的已读通知 |
| 过期通知清理 | NotificationCleanupTask | 每周日凌晨4点 | 清理90天前的所有通知（含未读） |
| 限流器缓存清理 | NotificationCleanupTask | 每小时 | 限流器缓存超过1000个时清理，防止内存泄漏 |
| 通知统计 | NotificationCleanupTask | 每天凌晨2点 | 统计前一天通知总数、未读数、阅读率 |
