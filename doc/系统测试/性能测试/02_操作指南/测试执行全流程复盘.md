## 📝 5. Phase 4 — JMeter 脚本开发（最复杂阶段）

### 5.1 目标
开发一份 4 个 Thread Group 的 JMX，参数化数据从 CSV 注入，断言确保业务流程闭环。

### 5.2 JMX 总体架构

```
exam-system-perf.jmx (28.7 KB)
├── Thread Group 1：登录性能基线
│   ├── CSV Data Set Config: students.csv (100 行)
│   ├── HTTP Sampler: POST /api/auth/login
│   ├── JSON Extractor: $.data.token → ${authToken}
│   └── Response Assertion: response_data contains "code":200
│
├── Thread Group 2：学生考试全流程（核心）
│   ├── CSV Data Set Config: students-exam.csv (43 行 班级 4)
│   ├── HTTP Sampler 1: POST /api/auth/login   → ${authToken}
│   ├── HTTP Sampler 2: GET /api/student/exam/my-exams
│   ├── HTTP Sampler 3: GET /api/student/exam/start/${examId}
│   ├── HTTP Sampler 4: POST /api/student/exam/auto-save
│   └── HTTP Sampler 5: POST /api/student/exam/submit
│
├── Thread Group 3：教师查询性能（默认禁用）
└── Thread Group 4：聊天消息压测（默认禁用）
```

### 5.3 PowerShell 脚本套件

| 脚本 | 用途 | 核心逻辑 |
|---|---|---|
| `start-perf-backend.ps1` | 启动 perf profile 后端 | 动态查找 `exam-system` 目录 + 三检查（MySQL/Redis/yml） |
| `verify-login.ps1` | 5 个账号登录可用性验证 | 调用 `/actuator/health` + `/api/auth/login` |
| `generate-csv.ps1` | 生成 4 个 CSV 数据集 | 查询 perf 库 → 写 CSV（剥离 BOM） |

### 5.4 生成的 4 个 CSV

| 文件 | 行数 | 用途 |
|---|---:|---|
| `students.csv` | 100 | TG1 登录基线 |
| `teachers.csv` | 15 | TG3 教师查询 |
| `students-exam.csv` | 43 | TG2 考试流程（班级 4 全部学生） |
| `chat-pairs.csv` | 30 | TG4 聊天对 |

### 5.5 真实踩坑记录（重点！）

> **以下 6 个 BUG 是 Phase 4 实战中真实遇到的，详见下面 §9 的根因分析与修复方案。**

1. ⚠️ **PowerShell 5 中文路径 ANSI 解码 bug** → 改用 ASCII 脚本
2. ⚠️ **CSV 文件 BOM 导致 JMeter 报错** → 用 .NET API 剥离 BOM
3. ⚠️ **JMeter ResponseAssertion 检测错误变量** → 改为检测 response_data
4. ⚠️ **examId=39 与学生 class_id 不匹配** → 重新生成 CSV
5. ⚠️ **JMeter 命令找不到** → 自动下载 JMeter 5.6.3 并配置 PATH
6. ⚠️ **mysql 命令在 PowerShell 报警告即崩溃** → 用 `$env:MYSQL_PWD` + `2>$null`

## 🐛 9. 全部 BUG 与修复合集（按 Phase 排序）

> **本节是 Phase 4 + Phase 6 真实踩过的 6 个坑的根因分析。每个 BUG 都按"症状 → 根因 → 修复 → 验证"四段式记录，便于复盘与故障排查。**

### BUG #1 — PowerShell 5 中文路径 ANSI 解码

- **Phase**：4
- **症状**：PowerShell 脚本里用 `$PSCommandPath` 推算项目目录时，中文字符 `在线考试系统` 被解码成 `鍦ㄧ嚎鑰冭瘯绯荤7缁`（GBK 双字节误读 UTF-8 三字节）
- **根因**：PowerShell 5（Windows Built-in）默认用 ANSI（GBK on 中文系统）解码 `.ps1` 文件
- **修复**：
  ```powershell
  # 方案 A：脚本内部用 ASCII 字符 + 动态查找项目根
  function Find-ExamSystemDir {
      $current = (Get-Location).Path
      for ($i = 0; $i -lt 5; $i++) {
          $candidate = Join-Path $current "exam-system"
          if (Test-Path $candidate -PathType Container) { return (Get-Item $candidate).FullName }
          $parent = Split-Path -Parent $current
          if ([string]::IsNullOrEmpty($parent) -or $parent -eq $current) { break }
          $current = $parent
      }
      return $null
  }
  ```
- **验证**：`start-perf-backend.ps1` 在中文路径下成功启动后端

### BUG #2 — CSV 文件 BOM 导致 JMeter 报错

- **Phase**：4
- **症状**：JMeter `IllegalArgumentException: Could not read file header line for file students.csv`
- **根因**：PowerShell 的 `Out-File -Encoding UTF8` 默认带 BOM（`EF BB BF` 三字节），JMeter 的 CSV Data Set Config 不支持 BOM
- **修复**：
  ```powershell
  # 用 .NET 直接读字节 → 剥离 BOM → 重新写
  $bytes = [System.IO.File]::ReadAllBytes($csvPath)
  if ($bytes.Length -ge 3 -and $bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
      $bytes = $bytes[3..($bytes.Length - 1)]
      [System.IO.File]::WriteAllBytes($csvPath, $bytes)
  }
  ```
- **验证**：JMeter 成功读取所有 4 个 CSV，315 个请求全部参数化正确

### BUG #3 — ResponseAssertion 检测错误变量

- **Phase**：4
- **症状**：⑤ POST `/api/student/exam/submit` 80% 失败，但 HTTP 状态码 200
- **根因**：`Assertion.test_field` 设为 `Assertion.variable` 检测 `${authToken}` 是否包含 "200"，但 token 是 JWT，里面没有 "200"
- **修复**：
  ```xml
  <!-- 改为检测 response_data 包含 "code":200 -->
  <ResponseAssertion>
    <stringProp name="-1166543264">&quot;code&quot;:200</stringProp>
    <stringProp name="Assertion.test_field">Assertion.response_data</stringProp>
    <intProp name="Assertion.test_type">16</intProp>  <!-- 16 = substring -->
  </ResponseAssertion>
  ```
- **验证**：43/43 请求全部断言通过

### BUG #4 — examId 与学生 class_id 不匹配

- **Phase**：4
- **症状**：TG2 学生考试流程报 `code=409 msg=您已完成该考试` 或 `不在该班级`
- **根因 1**：`students-exam.csv` 包含所有 perf 学生（300 人），但 examId=39 仅限 class_id=4（软件工程 2202）的 43 人
- **根因 2**：部分学生已有 `exam_record`，重复进入触发唯一索引冲突
- **修复**：
  ```sql
  -- 1. 清空 examId=39 的所有 record（仅 perf 用户）
  DELETE FROM exam_record
   WHERE exam_id = 39
     AND user_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_stu_%');

  -- 2. 重新生成 students-exam.csv，仅含 class_id=4 学生
  SELECT u.username FROM sys_user u
   WHERE u.username LIKE 'perf_stu_%'
     AND u.class_id = 4
     AND u.id NOT IN (SELECT user_id FROM exam_record WHERE exam_id = 39)
   ORDER BY u.username;
  ```
- **验证**：43 名 perf 学生全部成功完成考试闭环

### BUG #6 — JMeter HTML 报告中文模板乱码

- **Phase**：6
- **症状**：替换 mzky 中文模板后，生成的 HTML 标题显示 `JMeter鎬ц兘娴嬭瘯鎶ュ憡`
- **根因**：mzky 仓库的 `.fmkr` 文件是 UTF-8 编码，但 JMeter 在 Windows 中文系统用 `Charset.defaultCharset()` = GBK 解码，造成 UTF-8 字节流被 GBK 误解
- **修复**：
  ```powershell
  $gbk = [System.Text.Encoding]::GetEncoding(936)
  Get-ChildItem $rt -Recurse -Filter "*.fmkr" | ForEach-Object {
      $bytes = [System.IO.File]::ReadAllBytes($_.FullName)
      $contentUtf8 = [System.Text.Encoding]::UTF8.GetString($bytes)
      if ($contentUtf8 -match '性能|响应|请求|图表') {
          [System.IO.File]::WriteAllText($_.FullName, $contentUtf8, $gbk)
      }
  }
  ```
- **验证**：5 页 HTML 共 799 个中文字符全部正确渲染，标题显示「JMeter 性能测试报告」
