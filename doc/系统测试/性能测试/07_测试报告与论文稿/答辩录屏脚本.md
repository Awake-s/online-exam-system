# 答辩录屏脚本（30 秒视频，10 分钟可输出）

> **本文档定位**：一份「拍电影分镜本」式的逐步指南，照做即可录出一段 30 秒的答辩演示视频。
>
> **答辩用途**：把 JMeter 的"机器视角"翻译成"老师能看懂的真实学生体验"——5 个浏览器同时考试 + JMeter 同步压测 + 全部成功，**10 秒钟胜过 100 个 P95 数字**。
>
> **理论依据**：Apache JMeter Best Practices §16.7 明确规定 "Don't use 'View Results Tree' or 'View Results in Table' listeners during the load test"——压测期间的"业务可视化"必须依赖**外部录屏手段**，这是行业标准做法。

## 一、准备清单（5 分钟）

### 1.1 软件准备

| 软件 | 版本 | 来源 |
|---|---|---|
| **Windows 11 Snipping Tool** | 内置 | Win + Shift + S（自带录屏，22H2 起支持） |
| 备选：**OBS Studio** | 30.x | https://obsproject.com/ （专业但学习成本） |
| 备选：**Xbox Game Bar** | 内置 | Win + G（仅录单窗口，Win10/11 都自带） |

> **推荐**：用 Windows 11 自带的 **Snipping Tool** —— 毕设场景够用，零安装，输出 MP4。

### 1.2 环境准备（按顺序检查）

- [ ] **后端运行中**：终端 1 跑着 `start-perf-backend.ps1`，看到 `Started ExamSystemApplication`
- [ ] **前端运行中**：终端 2 跑着 `npm run dev`（`exam-web/`），http://localhost:5173 可访问
- [ ] **5 个 perf 学生账号可登录**：`perf_stu_001` ~ `perf_stu_005`，密码 `Test@123456`
- [ ] **JMeter 业务版 JMX 备好**：`04_JMeter脚本与数据/exam-system-perf-业务可读版.jmx`
- [ ] **屏幕分辨率 1920×1080 以上**：分屏后每个浏览器至少 380×500 像素

### 1.3 浏览器预备（关键！）

打开 **5 个 Chrome 无痕窗口**（Ctrl + Shift + N 五次），每个窗口预先：

1. 访问 http://localhost:5173/login
2. **不要登录**，等录屏开始才输入账号

> 💡 **为什么用无痕**：避免 Cookie 共享，5 个窗口=5 个独立用户。

## 三、30 秒分镜本（逐秒脚本）

| 时间 | 镜头内容 | 操作 |
|---|---|---|
| **0-3 秒** | **片头字幕** | 屏幕左上角加文字标题（Snipping Tool 不支持，可用剪映后期叠加）：<br>`在线考试系统 · 并发性能演示 · 43 名学生同时考试` |
| **3-6 秒** | **5 个窗口同时登录** | 在 5 个窗口里依次（快速）输入 `perf_stu_001~005` + `Test@123456`，**全部按下回车** |
| **6-10 秒** | **5 个窗口同时进考场** | 看到「我的考试」列表，**同时点击《Java 程序设计》期末考试 → 开始考试** |
| **10-15 秒** | **5 个窗口同时答题** | 在每个窗口里点选 1-2 道题的答案（哪个无所谓，演示用） |
| **15-20 秒** | **5 个窗口同时交卷** | **全部点击「交卷」按钮**，弹出确认框时全点「确定」 |
| **20-23 秒** | **5 个窗口同时显示"提交成功"** | 等所有窗口都显示成功提示——**这是高潮** |
| **23-27 秒** | **切到 JMeter 显示 0 错误统计** | 切到 JMeter GUI 或终端，显示 Aggregate Report 表格：43 个 sample / 0 errors / P95=147ms |
| **27-30 秒** | **片尾结论字幕** | 屏幕中央叠加：<br>`✓ 系统成功承载 43 学生并发考试`<br>`✓ 错误率 0.00% · APDEX 1.000 · 响应时间 P95 < 150ms` |

## 四、录屏操作步骤（Windows 11 Snipping Tool）

### 步骤 1：启动录屏

1. 按 **Win + Shift + S**
2. 工具栏切到 **录制（Record）** 图标（看起来像摄像机）
3. 拖选要录制的区域（建议**全屏 1920×1080**，方便后期裁剪）

### 步骤 2：执行 30 秒分镜（按上面的脚本走）

> ⚠️ **关键技巧**：不需要一次录完美。**录三遍取最好的一遍**。

### 步骤 3：保存视频

录制完成后点停止按钮 → 自动保存为 MP4 到 `视频/Screen Recordings/`，文件名形如 `Screen Recording 2026-04-29 230000.mp4`。

### 步骤 4：（可选）剪映后期

下载剪映（免费）：https://www.capcut.cn/

后期处理 5 分钟搞定：
- **片头字幕**（0-3 秒）：加纯白 / 黑底文字
- **配音**：用剪映的"文本朗读"功能（机器音免费），念以下文案：
  > "演示在线考试系统在 43 学生同时考试场景下的并发性能。可以看到，5 个浏览器同时登录、同时进入考场、同时答题、同时交卷，全部成功。JMeter 数据显示零错误率，APDEX 满分。"

## 六、答辩 PPT 嵌入方法

### 方法 A：直接嵌入（推荐）

PPT 中：**插入 → 视频 → 此设备上的视频** → 选 `答辩演示视频_43学生并发考试.mp4`

播放时机：
- **第 1 张 PPT 放完之后** ——开题先放视频，老师立刻有印象
- **第 6.3 节性能测试章节起头** ——讲性能测试时再放一遍强化

### 方法 B：超链接（如视频太大）

视频拷到 U 盘：`U盘\性能测试演示\43学生并发考试.mp4`  
PPT 里加超链接 → 选本地视频文件 → 答辩当场点击播放

## 八、答辩老师可能问的问题（提前准备）

| 问题 | 你的回答 |
|---|---|
| **这个视频是真录的吗？** | "是的，演示视频在本机环境录制，5 个浏览器是 5 个无痕窗口，分别用不同 perf 学生账号登录。" |
| **怎么证明这是并发不是串行？** | "5 个窗口同时显示"提交成功"是关键证据。同时 JMeter 还跑了 43 并发的版本，可见聚合报告 0 错误率。" |
| **为啥只录 5 个不录 43 个？** | "5 个是肉眼能看清的极限；43 个并发由 JMeter 同步压测验证，技术指标见 §6.3。视频和数据互相印证。" |
| **录屏算性能测试吗？** | "录屏是**业务功能验证**（User Acceptance Testing），不是性能测试本身。性能测试由 JMeter 完成，录屏只是把性能测试结果"翻译"成老师能直观看到的真实业务场景。" |
