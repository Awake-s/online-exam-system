## 4. 数据规模设计（**v2.0 校准**）

1. **真实度 95%+**：使用 Datafaker 生成中文姓名/手机/学号
2. **业务一致 100%**：所有数据走项目 Service 层，密码 BCrypt 加密、外键级联、审计字段自动填充
3. **规模合理**：300 学生 × 8 班 = 平均 38 人/班，严格贴合"小班 30-50 人"

## 5. 业务闭环压测场景设计

### 5.1 6 个核心场景（**[决策 D3]**）

| # | 代号 | 业务模拟 | 主要接口 | 线程范围 | 思考时间 | TPS 目标 | P95 目标 |

### 5.2 阶梯加压配置（JMeter Stepping Thread Group）

```
Step 1: 0→50 线程，2 分钟爬升
Step 2: 50→100，2 分钟爬升
Step 3: 100→150，2 分钟爬升
Step 4: 150→200，2 分钟爬升
每阶梯稳定：5 分钟
退化中止：错误率>1% 或 P95>3s
单场景耗时：~30 分钟
6 场景合计：~3 小时
```

### 5.3 通过标准

| 阶梯 | 通过标准 | 失败处理 |
|---|---|---|
| 50 并发 | **必须**：6 场景 P95 全部达标，错误率<0.1% | 触发系统调优 |
| 100 并发 | **建议**：至少 5 场景达标 | 记录为"已逼近上限" |
| 200 并发 | **找拐点即可** | 写入论文 7.2 展望章节 |

## 7. 🛡️ 安全保障三道防线（**v2.0 核心**）

### 防线 1：mysqldump 全库备份（**第一安全网**）

#### 7.1.1 备份命令（一键执行）

```powershell
# 在 Windows PowerShell 中执行（注意：必须用 cmd /c 字节级重定向，否则中文乱码）
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$backupFile = "d:\Java Projects\在线考试系统\doc\性能测试\backup\online_exam_system_$timestamp.sql"
New-Item -ItemType Directory -Force -Path "d:\Java Projects\在线考试系统\doc\性能测试\backup" | Out-Null
cmd /c "mysqldump -u root -p12345678 --default-character-set=utf8mb4 --single-transaction --routines --triggers online_exam_system > `"$backupFile`""
```

> **耗时**：当前数据库 ~5MB，备份 <5 秒。

#### 7.1.2 一键回退命令（万一有问题）

```powershell
# 选最近一次备份，秒级恢复
$latestBackup = Get-ChildItem "d:\Java Projects\在线考试系统\doc\性能测试\backup\*.sql" | Sort-Object LastWriteTime -Descending | Select-Object -First 1
cmd /c "mysql -u root -p12345678 --default-character-set=utf8mb4 online_exam_system < `"$($latestBackup.FullName)`""
```

> **承诺**：任何时刻都能秒级回到压测前的开发数据状态。

### 防线 2：测试数据固定前缀（**第二安全网**）

所有 v2.0 生成的用户数据用以下前缀，**保证与开发数据不混淆**：

| 前缀 | 用途 | 示例 |
|---|---|---|
| `perf_admin_` | 测试管理员 | perf_admin_01 |
| `perf_tea_` | 测试教师 | perf_tea_001 |
| `perf_stu_` | 测试学生 | perf_stu_001 ~ perf_stu_300 |

**一键清理脚本**（任何时刻执行都安全）：
```sql
-- 仅清理 perf_ 前缀生成的数据
DELETE FROM exam_answer WHERE record_id IN (SELECT id FROM exam_record WHERE user_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%'));
DELETE FROM exam_record WHERE user_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%');
DELETE FROM chat_message WHERE conversation_id IN (SELECT id FROM chat_conversation WHERE user1_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%') OR user2_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%'));
DELETE FROM chat_conversation WHERE user1_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%') OR user2_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%');
DELETE FROM sys_notification WHERE receiver_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%');
DELETE FROM teacher_class WHERE teacher_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%');
DELETE FROM teacher_subject WHERE teacher_id IN (SELECT id FROM sys_user WHERE username LIKE 'perf_%');
DELETE FROM sys_user WHERE username LIKE 'perf_%';
-- 注：试卷/题目/学科/班级等共享数据保留（也可单独按编号前缀清理）
```

### 防线 3：独立 schema（**最稳妥的可选项**）

**[决策 D8]**：是否使用独立 schema 隔离？

**方案 X（推荐，最稳妥）**：在 `online_exam_system_perf` 这个新 schema 中跑压测
```sql
CREATE DATABASE online_exam_system_perf 
  DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
-- 用 mysqldump 复制结构 + 全量数据
cmd /c "mysqldump -u root -p12345678 --default-character-set=utf8mb4 online_exam_system > tmp.sql"
cmd /c "mysql -u root -p12345678 --default-character-set=utf8mb4 online_exam_system_perf < tmp.sql"
-- 临时 application-perf.yml 切换 url 到 online_exam_system_perf
```

**优势**：
- 🛡️ **零风险**：开发库完全不动
- 🛡️ **可平行**：压测时仍能用开发库做功能测试

## 9. JMeter 脚本设计

### 9.1 脚本目录

```
doc/性能测试/jmeter-scripts/
├── 01_login_storm.jmx
├── 02_exam_start.jmx
├── 03_save_answer.jmx          ⭐ 最关键
├── 04_submit_storm.jmx
├── 05_marking.jmx
├── 06_chat_fanout.jmx
└── shared/
    ├── login_token_provider.groovy   # JSR223 PreProcessor：动态登录获取 JWT
    └── csv_users.csv                  # 用户名+密码 CSV，供 CSV Data Set Config 读取
```

### 9.2 关键脚本要点

每个 .jmx 包含：
1. **CSV Data Set Config** —— 读 `perf_stu_001~perf_stu_300` 用户名/密码
2. **HTTP Request Defaults** —— 默认 protocol=http, host=localhost, port=8080
3. **JSR223 PreProcessor (Groovy)** —— 在每线程开始时调用 `/api/auth/login` 获取 JWT 存到线程变量
4. **HTTP Cookie Manager** —— 自动管理 Cookie
5. **HTTP Header Manager** —— `Authorization: Bearer ${jwt_token}`
6. **Stepping Thread Group** —— 阶梯加压
7. **Aggregate Report Listener** —— 输出 TPS/P95/P99
8. **Backend Listener (HTML Dashboard)** —— 生成最终 HTML 报告
9. **Response Assertion** —— 校验 `code:0`（项目 Result.success）

### 9.3 一键执行命令

```powershell
$jmeter = "D:\Tools\apache-jmeter-5.6.3\bin\jmeter.bat"
$scripts = "d:\Java Projects\在线考试系统\doc\性能测试\jmeter-scripts"
$results = "d:\Java Projects\在线考试系统\doc\性能测试\results"
New-Item -ItemType Directory -Force -Path $results | Out-Null

# 单场景跑（以 S3 为例）
& $jmeter -n -t "$scripts\03_save_answer.jmx" `
    -l "$results\03_save_answer_$(Get-Date -Format 'yyyyMMdd_HHmmss').jtl" `
    -e -o "$results\03_save_answer_dashboard"

# 6 场景批量跑
1..6 | ForEach-Object {
    $script = Get-ChildItem "$scripts\0$_*.jmx" | Select-Object -First 1
    & $jmeter -n -t $script.FullName -l "$results\$($script.BaseName).jtl" -e -o "$results\$($script.BaseName)_dashboard"
}
```
