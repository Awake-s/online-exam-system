## 四、自动化审计机制（v2 核心创新）

### 4.1 审计能力

`audit-collection.ps1` 每次提交前自动校验：

1. **URL 真实性**：每个用例的 URL 必须在 19 个 Controller 中实际存在
2. **环境隔离**：环境变量必须使用 it_xxx 测试账号（不能用业务账号）
3. **统计指标**：自动计算用例数 / 断言数 / 反向占比
4. **评级机制**：A+（0 issues + 反向 ≥ 30%）/ A（无 issue 但反向不足）/ B（有 issue）


**收益**：v1 中 3 个被跳过的用例（GET /paper/{id}, /marking/list, /score/class）现在全部可执行。

### 5.3 硬编码 bug（已修复）

```diff
- pm.test('用户名 = admin', () => pm.expect(json.data.username).to.equal('admin'));

```

**收益**：用例对环境变量友好，更换账号时不会失败。

### 5.4 错误密码用例（已增强）

```diff
- pm.test('错误信息含「用户名或密码错误」', () => pm.expect(json.message).to.include('用户名或密码错误'));
+ pm.test('错误信息属于合法失败消息', () => {
+   const m = json.message;
+   const valid = m.includes('用户名或密码错误') || m.includes('登录失败次数过多');
+   pm.expect(valid, 'message: ' + m).to.be.true;
+ });
```

**收益**：兼容 LoginRateLimiter 触发后的限流消息（源码依据 `@AuthServiceImpl.java:72`）。

## 六、严格基于源码追溯（v2.0 核心承诺）

每个新增用例的 `event.test.script.exec` 都包含 `// 源码: @文件:行号` 注释。例如：

```javascript
// 源码: @QuestionController.java @PreAuthorize 仅 TEACHER/ADMIN
// → AccessDeniedException → GlobalExceptionHandler.java:84-87 → HTTP 200 + code=403
pm.test('HTTP 200', () => pm.response.to.have.status(200));
pm.test('业务码 403 (越权)', () => pm.expect(pm.response.json().code).to.equal(403));
```

可用 audit-collection.ps1 自动校验，**任何无源码追溯的断言不能合入**。
