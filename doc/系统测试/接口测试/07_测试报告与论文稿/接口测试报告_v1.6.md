### 4.3 未覆盖范围声明 (Test Items Not Executed)

按 ISO/IEC/IEEE 29119-3:2021 §8.2.6 *Test Items Not Executed* 条款，本测试集存在以下**协议层未覆盖范围**，已诚实披露：

| # | 未覆盖项 | 协议 | 端点 | 未覆盖原因 | 缓解措施 | 影响评估 |
|---:|---|---|---|---|---|---|
| NE-001 | `ChatTypingController.handleTyping` | **STOMP over WebSocket** | `@MessageMapping("/chat/typing")` | Postman/Newman 协议层不支持 STOMP 帧（CONNECT/SUBSCRIBE/SEND/MESSAGE）；本测试集统一采用 HTTP/1.1 + JSON 协议栈 | ① 前端 `art-chat-window/index.vue` E2E 集成验证（CHAT-308 / CHAT-325 系列回归用例）；② STOMP 通道由 Spring `WebSocketStompClient` 异步集成测试可补充 | 低 — 该端点为 disposable typing 事件，丢包可接受（前端有 TTL 兜底），非关键业务路径 |

**未覆盖端点占比**：1 / (108 REST + 1 STOMP) = **0.92%**，REST 协议范围内覆盖率 **108/108 = 100%**。

**改进路线**：v1.7 可引入 `WebSocketStompClient` 集成测试覆盖 STOMP 通道（预计工作量 2 小时），但鉴于该端点已通过前端 E2E 间接验证且无关键业务流，**v1.6 阶段维持现状**。

## 十、经验教训 (Lessons Learned)

### 10.1 成功经验

1. **双工具交叉验证机制**: 任何单一工具的统计偏差都不能影响最终结论，提升测试结果权威性
2. **错误推测大规模应用**: 84.8% 反向用例占比远超基线，暴露安全设计盲点效果显著
3. **源码追溯审计**: 每条用例都基于 Controller + GlobalExceptionHandler + SecurityConfig 源码反推，断言精确
4. **自动化文档生成**: PowerShell 脚本将 collection 转为人类可读规范，避免重复劳动

### 10.2 改进经验

1. **Apifox 契约测试**: 默认开启会与 Postman v2.1 格式产生显示偏差，需主动关闭
2. **测试集与接口定义分离**: Postman v2.1 不携带响应定义，长期维护建议迁移 OpenAPI 3.0
3. **CI/CD 集成**: 当前为本地手动执行，下一步集成到 GitHub Actions / GitLab CI
4. **专业测试账号**: `it_*` 前缀账号有效避免污染生产业务数据
