## 三、过程性"伪缺陷"记录与处置

> 以下问题不是被测系统（SUT）缺陷，但在测试过程中暴露了**测试工具/测试集本身**的问题，按 ISO/IEC/IEEE 29119-3 §8.2.7 应当记录处置方式。

### 伪缺陷 #1：Apifox UI 显示"108 用例失败"

| 字段 | 内容 |
|---|---|
| 报告时间 | 2026-04-30 19:30 |
| 严重度 | ⚠️ 误判（不是真缺陷） |
| 优先级 | High（影响展示） |
| 状态 | ✅ 已修复 |
| 现象描述 | Apifox 自动化测试报告显示"通过 37 / 失败 108 (74.48%)"，但 Newman CLI 跑同一份 collection 显示 100% 通过 |
| 根本原因 | Apifox 默认开启"契约测试"功能，校验实际响应 HTTP 状态码 = 接口文档定义；而 Postman v2.1 collection 不携带 OpenAPI 响应定义，Apifox 默认生成 200 响应。反向用例返回 401 时被契约测试判定为"不一致" |
| 证据链 | 1. [Apifox 官方文档：契约测试默认启用](https://docs.apifox.com/contract-testing) 2. Postman v2.1 collection 实测 0 个 responses 字段 3. 同一份 collection 在 Newman 中 100% 通过 |
| 修复方案 | Apifox 项目设置 → 校验响应设置 → 关闭"自动化测试步骤校验响应"开关 |
| 修复验证 | ✅ 修复后 145/145 全部绿色通过 |
| 修复文档 | `02_操作指南/Apifox_红色失败修复指南.md` |

### 伪缺陷 #2：测试集 Postman v2.1 缺失 OpenAPI 响应定义

| 字段 | 内容 |
|---|---|
| 报告时间 | 2026-04-30 19:30 |
| 严重度 | ℹ️ Postman v2.1 格式特性，非 bug |
| 优先级 | Low（影响 Apifox 契约测试模式） |
| 状态 | ✅ 已通过工具开关绕过 |
| 现象描述 | Postman v2.1 collection 不含每个接口的 401/403/404 响应定义 |
| 根本原因 | Postman v2.1 规范本身不强制要求响应定义；本测试集设计时使用 PowerShell 脚本生成，未填充示例响应 |
| 后续优化 | （v2.0 改进项）将测试集迁移至 OpenAPI 3.0 + Apifox 原生格式，为每个接口添加 200/401/403 响应定义 |
