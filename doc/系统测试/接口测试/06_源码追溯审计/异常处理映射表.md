### 2.5 认证授权异常（关键 ⭐）

#### 2.5.1 未登录（无 Token / Token 无效）

> **触发链路**：JWT 过滤器 → 没有合法 Token → SecurityContext 无 Authentication → `.anyRequest().authenticated()` 拒绝 → SecurityConfig 的 `authenticationEntryPoint`

| 维度 | 值 | 源码 |
|---|---|---|
| HTTP 状态 | **401 Unauthorized** | `SecurityConfig.java:61` `resp.setStatus(SC_UNAUTHORIZED)` |
| 响应体 code | **401** | `SecurityConfig.java:63` `r.put("code", 401)` |
| 响应体 message | "未登录或Token已过期" | `SecurityConfig.java:64` |

> **注意**：这是**真 HTTP 401**（过滤器层直接 setStatus），**不经过** `@RestControllerAdvice`。

> **测试断言模板**：
> ```javascript
> pm.test('HTTP 401', () => pm.response.to.have.status(401));
> // 或者也可以同时校验 body
> pm.test('业务码 401', () => pm.expect(pm.response.json().code).to.equal(401));
> ```

#### 2.5.2 角色越权（@PreAuthorize 拒绝）

> **触发链路**：JWT 过滤器 → 解析 Token 成功，SecurityContext 设置 ROLE_X → Controller 方法的 `@PreAuthorize("hasRole('Y')")` 检查失败 → 抛 `AccessDeniedException` → **被 GlobalExceptionHandler 拦截** → `Result.error(403, "无权限访问")`

| 维度 | 值 | 源码 |
|---|---|---|
| HTTP 状态 | **200 OK** | 由 `Result.error()` 返回，不主动 setStatus |
| 响应体 code | **403** | `GlobalExceptionHandler.java:86` `Result.error(403, "无权限访问")` |
| 响应体 message | "无权限访问" | 同上 |

> **重要**：此处 HTTP 是 200，**不是 403**！这与 SecurityConfig 的 `accessDeniedHandler`（HTTP 403）不同。

> **测试断言模板**：
> ```javascript
> pm.test('HTTP 200', () => pm.response.to.have.status(200));
> pm.test('业务码 403 (无权限)', () => pm.expect(json.code).to.equal(403));
> pm.test('错误信息含「无权限」', () => pm.expect(json.message).to.include('无权限'));
> ```

#### 2.5.3 SecurityConfig 的 accessDeniedHandler（极少触发）

> **触发链路**：在 SecurityFilterChain 配置层（如 antMatchers + hasRole）拒绝。本项目主要用 `@PreAuthorize` 而非链式配置，所以**这条路径基本不触发**。

| 维度 | 值 | 源码 |
|---|---|---|
| HTTP 状态 | **403 Forbidden** | `SecurityConfig.java:69` `resp.setStatus(SC_FORBIDDEN)` |
| 响应体 code | **403** | `SecurityConfig.java:71` |
| 响应体 message | "无权限访问" | `SecurityConfig.java:72` |

> **判断口诀**：
> - 想知道是哪条路径？看 HTTP 状态：**403 → SecurityConfig**；**200 + code=403 → GlobalExceptionHandler**

## 4. 接口测试断言决策树

```
┌─ 接口能否登录前调用？─── 是 ──→ 公开接口（如 /captcha, /login）
│                              ├── 正向: HTTP 200, code=200
│                              ├── 业务异常: HTTP 200, code=409 (BusinessException)
│                              └── 参数错误: HTTP 200, code=400
│
└── 否（需登录）
    ├─ 用合法 Token 调 ─────── 正向: HTTP 200, code=200
    ├─ 不带 Token 调 ─────────── HTTP 401, code=401
    ├─ 带错误 Token 调 ──────── HTTP 401, code=401
    ├─ 角色匹配但越权 ───────── HTTP 200, code=403  ⭐ 最易出错
    └─ 业务异常（如 ID 不存在）─ HTTP 200, code=409
```
