## 1. 三层工具链定位

本项目接口测试采用**「三层架构」**，是经过深思熟虑的工程化方案：

```
┌─────────────────────────────────────────────────────────┐
│  第三层：自动化执行层                                       │
│  Newman v6.2.2 + htmlextra Reporter（命令行 CLI）          │
│  • 用途：CI/CD、批量回归、生成 HTML 报告                    │
│  • 调用：run-newman-test.ps1 一键脚本                       │
└─────────────────────────────────────────────────────────┘
                          ↑
                          │ 共享同一份 collection.json
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第二层：调试与演示层                                        │
│  Apifox 2.x（国产 GUI / 中文界面 / 信创合规）                │
│  • 用途：单接口调试、答辩演示、用例可视化编辑                  │
│  • 操作：Apifox实战手册v1.6.md（导入流程已成熟）             │
└─────────────────────────────────────────────────────────┘
                          ↑
                          │ 100% 兼容 Postman v2.1 标准
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第一层：数据格式层                                          │
│  Postman Collection v2.1 标准（国际通用 JSON Schema）        │
│  • 文件：exam-system.postman_collection.json                │
│  • 文件：exam-system.postman_environment.json               │
│  • 性质：全球开放标准（schema.getpostman.com）               │
└─────────────────────────────────────────────────────────┘
```

## 2. 三个工具的关系（澄清常见误解）

### 2.1 ❌ 误解："项目用 Postman 不是 Apifox"

> **真相**：项目**主用 Apifox（GUI）+ Newman（CLI）**，**根本没装 Postman 桌面客户端**。"Postman" 这个词在本项目里**只出现在两个地方**：
> 1. JSON 文件后缀 `.postman_collection.json` —— 这是**国际数据格式标准的命名约定**，类似 `.xlsx` 不代表必须用 Office Excel。
> 2. Newman —— 这是**Postman 公司开源的官方命令行工具**，**Apifox 完全兼容 Newman**，业内通用。

### 2.2 ❌ 误解："Apifox 只是 Postman 的山寨版"

> **真相**：Apifox 是**国内 API 工具市场份额第一**（2024-2026 数据），被字节跳动、阿里巴巴、腾讯、华为等头部企业广泛采用，是**信创战略导向**的国产替代品。它**完全兼容 Postman Collection v2.1 国际标准**，反而支持更丰富的协作、文档、Mock 等功能。

### 2.3 ❌ 误解："换工具就要重写测试集"

> **真相**：Apifox 和 Newman **共享同一份 `.postman_collection.json` 文件**，**零迁移成本**。这是采用国际标准带来的最大工程红利 — 工具可换、数据不变。

## 4. 论文/答辩标准话术

### 4.1 短版（30 秒，论文摘要 / 演讲开场）

> 本系统采用 **Apifox（国产专业 API 协作平台，国内市场份额第一）** 进行接口测试用例的可视化编辑与单接口调试，
> 同时引入 **Newman（Postman 官方命令行工具）** 实现自动化批量执行与 HTML 报告生成，
> 两者**共享 Postman Collection v2.1 国际标准**作为底层数据格式，
> 形成 **「GUI 调试 + CLI 自动化」双轨工具链**，符合**信创战略导向**与**工程化最佳实践**。

### 4.2 中版（论文 4.4 章节首段）

> 接口测试是验证后端 RESTful API 正确性的关键环节。本系统选用 **Apifox 2.x** 作为主测试工具，
> 它是国产专业 API 协作平台，2024-2026 年国内市场份额第一，
> 被字节跳动、阿里、腾讯等头部企业广泛采用，**完全兼容 Postman Collection v2.1 国际标准**。
> 
> 同时保留 **Newman v6.2.2** 命令行工具实现自动化批量执行，
> 通过 **htmlextra Reporter** 生成中文 HTML 测试报告，便于答辩展示与论文附录。
> 
> 测试集采用 **Postman Collection v2.1 JSON 格式**存储，**Apifox 与 Newman 共享同一份配置**，
> 实现 **GUI 调试与 CLI 自动化的零成本协同**，符合工程化最佳实践与信创战略导向。

### 4.3 长版（论文 4.4.1 工具选型对比）

| 工具 | 角色 | 优势 | 在本项目的用途 |
|---|---|---|---|
| **Apifox** | 主用 GUI | 中文、信创、国内第一、协作能力强 | 用例编辑、单接口调试、答辩演示 |
| **Newman** | CLI 自动化 | 命令行、CI/CD 友好、官方维护 | 批量执行、HTML 报告 |

## 5. 与原始命名约定的兼容

为保持 **Postman v2.1 国际标准**的"格式中立性"，本项目所有 JSON 文件保留**国际惯用命名**：
- `exam-system.postman_collection.json`
- `exam-system.postman_environment.json`

这种命名是**全球开发者社区共识**：
- Apifox 官方导出也用此后缀
- Newman 官方文档示例也用此后缀
- GitHub 上 90%+ 开源项目用此后缀

**结论**：文件名带 `postman_` 字样**不代表必须用 Postman 软件**，正如 `.xlsx` 文件不代表必须用 Office Excel（WPS 也能打开）。

## 6. 验证：Apifox 能否跑当前测试集？

### 6.1 一键验证步骤

参考 `02_操作指南/Apifox实战手册v1.6.md` 的“导入 Collection”章节：

1. 打开 Apifox → 新建项目"在线考试系统接口测试"
2. 项目设置 → 导入 → 数据格式选 **"Postman"**
3. 选择文件：`doc/接口测试/04_Postman脚本与数据/exam-system.postman_collection.json`
4. 同样方式导入环境变量 `exam-system.postman_environment.json`
5. 顶部环境选择器选 `exam-system-dev`
6. 进入 **"自动化测试"** 标签 → 新建场景 → 导入全部 49 个接口 → 运行

**期望结果**（与 Newman 完全一致）：
- 49 个 Request 全部 200/401/200 状态码正确
- 120 个 Assertion 全部 ✓ 通过
- 总耗时约 4-5 秒

### 6.2 故障排查

| 问题 | 原因 | 解决 |
|---|---|---|
| `{{baseUrl}}` 红色（未识别） | 没选环境 | 顶部环境选 `exam-system-dev` |
| 401 Unauthorized | Setup 没先跑 | 必须先按顺序跑 Setup 文件夹 |
| Token 不刷新 | 环境变量未保存 | 检查 Setup-1/2/3 的 Tests 脚本是否执行 |
| 接口找不到 | 后端没启动 | `cd exam-system && mvn spring-boot:run` |

## 7. 维护准则

1. **永不删除 `.postman_*` 后缀** — 这是国际标准命名约定，删除等于失去跨工具能力。
2. **始终称呼 "Apifox + Newman 双轨"** — 这是项目主用工具的正确表述。
3. **测试用例的源代码是 JSON** — Apifox / Newman 都是渲染工具，JSON 才是源代码（参考"基础设施即代码"理念）。
4. **变更后必须跑 audit-collection.ps1** — 自动化审计是工具链的"质量基线守门员"。
