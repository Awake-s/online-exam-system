## 3. 单接口调试演示（答辩演示重点 · 5 分钟）

**演示场景**：向老师展示「我们如何精确测试一个接口」。  
**推荐演示用例**：`📋 Setup` → `✅ Setup-1 · ADMIN 登录获取 Token`（最经典、最直观）

### 3.1 操作步骤

1. 左侧 → 展开 **`📋 Setup · 前置准备`** → 点击 **`✅ Setup-1 · ADMIN 登录获取 Token`**
2. 中间区域显示请求详情：
   - **方法**：`POST`
   - **URL**：`{{baseUrl}}/api/auth/login`
   - **Body** 标签：JSON 内容（自动加载）
     ```json
     {
       "username": "{{adminUser}}",
       "password": "{{adminPass}}"
     }
     ```
3. **切换到「后置操作」标签**（Apifox 等同 Postman 的 Tests 标签）
4. 你会看到答辩演示的关键代码：
   ```javascript
   const json = pm.response.json();
   pm.test('HTTP 状态码为 200', () => pm.response.to.have.status(200));
   pm.test('业务码 code=0 (成功)', () => pm.expect(json.code).to.equal(0));
   pm.test('返回 token 字段', () => pm.expect(json.data.token).to.be.a('string').and.not.empty);
   pm.test('返回 userInfo.roleCode = ADMIN', () => pm.expect(json.data.userInfo.roleCode).to.equal('ADMIN'));
   pm.test('响应时间 < 1000ms', () => pm.expect(pm.response.responseTime).to.be.below(1000));
   pm.environment.set('adminToken', json.data.token);
   ```
5. 点击右上角 **「发送」** 按钮
6. **下方查看 3 个标签**：
   - **响应**：JSON 数据（含 token 字段）
   - **后置操作结果**：5 个 ✓ 绿色断言（这是给老师看的关键）
   - **响应时间**：通常 100-300ms

### 3.2 推荐演示的另外 3 个用例（建立完整说服力）

| 用例 | 演示价值 | 路径 |
|---|---|---|
| `Setup-1 · ADMIN 登录` | **正向 / 鉴权链路** | 📋 Setup |
| `1️⃣ 认证模块` → 错误密码 | **反向 / 业务异常** | 1️⃣ |
| `3️⃣ 用户管理` → 学生越权 | **反向 / 越权 (403)** | 3️⃣ |
| `🔐 ChatController` → 任一未登录 | **反向 / 鉴权 (401)** | 12 |

## 7. 故障排查 FAQ

### Q1: Apifox 导入后 folder 数量 ≠ 21？
**A**: 你导入了旧版本 Collection。重新导出 v1.6 文件，确认文件大小约 380KB+，145 个用例。

### Q2: `{{baseUrl}}` 显示红色（未识别）？
**A**: 顶部环境选择器没选 `exam-system-dev`。

### Q3: 单接口跑通但批量跑失败？
**A**: 批量跑没按顺序导入用例。**Setup 必须最先**，**Cleanup 必须最后**。重新组织自动化测试场景的步骤顺序。

### Q4: 鉴权扩展 folder 里的用例返回 200 而非 401？
**A**: 检查请求是否带了 Token — 这些用例是**未登录用例**，应该没有 Authorization 头。检查 Setup 是否在它们之前跑了（导致全局 Token 已设置）。**实际不会有问题** — 因为这些用例的 request.header 都是空的，没有 Authorization。

### Q5: 错误密码用例的"用户名或密码错误"断言失败？
**A**: 后端有 LoginRateLimiter，触发后会返回"登录失败次数过多"。Collection v1.6 已兼容这两种消息（参见 `@AuthServiceImpl.java:72`），属于**预期内容**。

### Q6: HTML 报告中文乱码？
**A**: 用 Chrome 或 Edge 打开。htmlextra Reporter 的 HTML 是 UTF-8 编码。

### Q7: Apifox 登录页面挡住？
**A**: 点 **"暂不登录"** 或 **"游客身份使用"**，免费版无任何功能限制。

### Q8: Apifox 与 Postman 的术语对应？

| Apifox（中文） | Postman（英文） |
|---|---|
| 后置操作 | Tests |
| 自动化测试 | Runner |
| 发送 | Send |
| 接口 | Request |
| 测试套件 | Collection |
| 环境管理 | Environments |
